## 1. GA4 계정 구조 및 필수 설정 (48시간 내 완료)

### 1.1 계정/속성 구성
```
계정명: 소상공인마케팅도구
├─ 속성1: 메인사이트_GA4 (측정ID: G-XXXXXXXXX)
│  ├─ 데이터스트림: 웹 (도메인: yoursite.com)
│  ├─ 보고시간대: Asia/Seoul

│  └─ 산업군: 온라인 커뮤니티 > 전문 서비스
└─ 속성2: 도구페이지_별도추적 (선택사항, 도구별 깊이 분석용)
```

**산업군 선택 이유**: "전문 서비스" = 콘텐츠+도구 하이브리드 모델 최적화. Google 자동 리포트에서 '리드 생성(도구 사용)', '참여도(체류시간)' 지표 우선 표시[1][4].

## 2. 핵심 이벤트 설정 (광고 수익 사이트 특화)

### 2.1 자동 수집 이벤트 (기본 활성화)
| 이벤트명 | 발생 조건 | 수익 연관성 | 목표값 |
|---------|----------|------------|--------|
| **page_view** | 모든 페이지 로드 | 직접 (Impression 증가) | 세션당 3.5 페이지[5] |
| **scroll** | 90% 스크롤 | 간접 (광고 노출 완료) | 도구 페이지 60%+[3] |
| **user_engagement** | 10초+ 체류 | 간접 (AdSense 시간 가중치) | 평균 4분[5] |

### 2.2 커스텀 이벤트 (GTM에서 생성) — **핵심 차별화**
```javascript
// 이벤트 1: 도구 사용 (QR 생성기 예시)
dataLayer.push({
  'event': 'tool_used',
  'tool_name': 'qr_code_generator',
  'user_type': 'returning', // 신규/재방문
  'session_duration': '320' // 초 단위
});

// 이벤트 2: 템플릿 다운로드
dataLayer.push({
  'event': 'template_download',
  'template_type': 'invoice',
  'format': 'excel'
});

// 이벤트 3: 광고 클릭 (AdSense 자동 추적 보완)
dataLayer.push({
  'event': 'ad_interaction',
  'ad_unit_id': 'sidebar-1',
  'page_rpm_tier': 'high' // RPM $5+ 페이지 태그
});
```

**GTM 트리거 설정**:
- `tool_used`: 버튼 클릭 (ID: #generate-qr) → 맞춤 이벤트 전송
- `template_download`: 다운로드 링크 클릭 (URL 포함: /download/) → file_download 이벤트[3]
- `ad_interaction`: .adsense-unit 클래스 클릭 → 맞춤 HTML 태그로 push

### 2.3 전환 이벤트 등록 (GA4 관리 > 전환)
**최우선 전환 3개** (주간 리포트 핵심 지표):
1. **tool_used** → 목표: 주간 1,500회 (도구 페이지 트래픽의 20%)
2. **template_download** → 목표: 주간 800회 (콘텐츠 페이지 CTR 12%)
3. **scroll (90%)** → 목표: 도구 페이지 60%, 글 페이지 40%

**근거**: Scroll 90% = 광고 4-5개 노출 완료 = RPM 30% 증가 상관관계[5].

## 3. 맞춤 측정기준 (Custom Dimensions) — **팀 KPI 추적용**

### 3.1 사용자 속성 (User Property)
| 속성명 | 설정 방법 | 활용 목표 |
|--------|----------|----------|
| **user_segment** | 첫 방문 페이지 기준 분류 (도구/블로그/리스트) | 도구 사용자 재방문율 70%+ 유지[1] |
| **traffic_source_detail** | UTM 태그 (정우 제공 키워드 매핑) | "free qr code" 키워드 → 도구 전환율 18%+ |
| **subscription_status** | 쿠키 기반 (뉴스레터 가입 여부) | 가입자 세션 시간 2배 확인 |

**GTM 변수 설정**:
```javascript
// 사용자 속성 전송 (페이지 로드 시)
gtag('set', 'user_properties', {
  user_segment: '{{First Page Type}}', // 도구/블로그/리스트
  traffic_source_detail: '{{URL Query - utm_term}}'
});
```

### 3.2 이벤트 매개변수 (Event Parameter)
| 매개변수명 | 연결 이벤트 | 데이터 예시 | 분석 목적 |
|-----------|------------|------------|----------|
| **page_category** | page_view | "도구/QR생성기" | 카테고리별 RPM 비교 (도구 $8, 글 $3)[5] |
| **content_author** | page_view | "하윤" | 작성자별 평균 체류시간 (하윤 4.2분, 목표 4분+) |
| **keyword_rank** | page_view | "1-3위" | 상위 랭크 페이지 이탈률 (1-3위 28%, 4-10위 45%) |

**설정 경로**: GA4 관리 > 맞춤 정의 > 맞춤 측정기준 만들기
- 범위: 이벤트
- 매개변수: `page_category`
- 표시 이름: 페이지 카테고리

## 4. AdSense 수익 추적 통합

### 4.1 AdSense와 GA4 연결 (24시간 내 완료)
**절차**:
1. AdSense 계정 > 설정 > 액세스 및 승인 > Google 애널리틱스 통합
2. GA4 속성 ID 입력 (G-XXXXXXXXX)
3. "AdSense 데이터 가져오기" 활성화

**연결 후 확인사항**:
- GA4 > 수익 창출 > 게시자 광고 → 노출수, 클릭수, 수익 표시 (48시간 후)[4]
- 맞춤 보고서에서 `이벤트 이름 = ad_click` + `AdSense 수익` 교차 분석 가능

### 4.2 RPM 추적 계산 필드 (GA4 탐색 분석)
**수식**:
```
RPM = (AdSense 수익 / 페이지뷰) × 1000
```

**탐색 분석 설정**:
- 측정기준: 페이지 제목 및 화면 클래스, 페이지 카테고리
- 측정항목: 조회수, AdSense 수익, 계산 필드(RPM)
- 필터: 조회수 > 100 (통계적 유의성)

**목표 RPM** (키워드 리포트 기반):
- 도구 페이지: $8+ (QR 생성기, 계산기)
- 가이드 글: $4-6 ("how to" 키워드)
- 리스트 글: $3-5 (템플릿, 체크리스트)
