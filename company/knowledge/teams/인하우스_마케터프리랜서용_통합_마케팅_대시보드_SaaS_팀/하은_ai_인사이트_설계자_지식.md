
### 마케팅 KPI 이상 감지(anomaly detection) 알고리즘 비교: Z-score vs Prophet vs LSTM 광고 데이터 적용 사례
# 마케팅 KPI 이상 감지 알고리즘 비교: 광고 데이터 실전 가이드

## 핵심 알고리즘 비교

| 알고리즘 | 작동 원리 | 광고 데이터 적용 | 실시간 처리 | 정확도 |
|---------|---------|--------------|----------|-------|
| **Z-Score** | 표준편차 기반 평균 편차 탐지[5] | 단순 이상치(스파이크) 탐지 | 즉시 | 낮음 |
| **Prophet** | 트렌드·계절성 분해 후 잔차 분석[4] | 정적 데이터, 장기 추세 예측 | 느림 | 중간 |
| **LSTM** | 시계열 장기 의존성 학습[4][5] | 동적 데이터, 단·장기 패턴 | 중간-느림 | 높음 |

---

## 알고리즘별 특성 & 적용 시나리오

### 1. **Z-Score (저수준 이상치 탐지)**

**작동 방식**
- 평균에서 표준편차의 N배 이상 벗어난 값을 이상치로 판단[5]
- 예: CTR 평균 4%, 표준편차 0.5%일 때 → Z-Score > 2 = 비정상

**광고 데이터 사용 사례**
- 갑작스러운 CPC 급등/급락
- 일일 광고 지출 이상 폭증
- 클릭 수 비정상적 저하

**한계**
- 계절성/트렌드를 무시 → 성수기 정상값도 이상치로 오분류
- 점진적 변화 감지 불가

---

### 2. **Prophet (구조적 패턴 기반)**

**작동 방식**
- 트렌드 + 계절성 + 휴일 효과 분해[4]
- 추세와 계절성 제거 후 남은 잔차에서 이상치 탐지[5]

**광고 데이터 사용 사례**
- 요일별/월별 검색량 변동성 큰 데이터 처리[5]
- 명절·이벤트 기간 이상 마케팅 효과 분석
- 신규 캠페인 성과 예측 (정적 데이터)

**강점**
- 직관적 시각화로 임원진 보고 용이
- 입문 난이도 낮음 (빠른 모델 개발)
- 이슈 트래킹, 트렌드 예측에 우수[4]

**한계**
- 실시간 예측 부적합 (대량 데이터 처리 속도 느림)[4]
- 동적(변동 심한) 광고 데이터에 성능 저하[4]

---

### 3. **LSTM (딥러닝 패턴 학습)**

**작동 방식**
- RNN 기반 시계열 장기 의존성 자동 학습
- 과거 n시간/일 데이터로 미래 값 예측 후 잔차로 이상 판단[4][5]

**광고 데이터 사용 사례**
- 실시간 거

### **GA4·메타·네이버SA 채널 통합 기여 모델 프레임워크**
**GA4를 중앙 허브로 Meta/TikTok 비용·클릭 데이터 임포트 + 데이터 기반 어트리뷰션(ML 동적 할당) 적용. 네이버SA는 GA4 이벤트 트래킹으로 통합, Looker Studio/BigQuery로 크로스채널 ROAS 계산. 차이: Meta(클릭7d/뷰1d 100% 할당) vs GA4(멀티터치 분배).** [1][4][5]

#### **1. 데이터 수집 통합 (Data Import + GCLID/파라미터)**
- **Meta/TikTok**: GA4 관리 > 데이터 가져오기 > 커넥터 생성 → 매일 자동 임포트(비용/클릭/노출). GA4 비용 보고서에서 GA4 전환(매출/구매)과 조인[4].
  - **차이 보정**: Meta 100% vs GA4 분배 → GA4에서 Meta 기여도 40% 예시(유튜브50%/구글20%)[5].
- **네이버SA**: UTM 파라미터(gtag/GA4 이벤트)로 GA4 세션 태그 → 네이버 터치포인트 GA4 기여 보고서에 자동 반영[1][2].
- **GA4 설정**: 관리 > 기여 분석 > **데이터 기반 모델 선택**(ML로 터치 순서/상호작용 분석, 전환 가치 동적 분배)[1][2].
  - **예시 분배**: Google20%/Meta30%/이메일50% (시간 감쇠 적용)[1].

2. **임포트 연결**: Meta/TikTok 커넥터 ON → 비용 보고서 확인(GA+Meta ROAS 통합)[4].
3. **모델 선택**: 기여 분석 > **데이터 기반**(ML: 첫40%/중간20%/마지막40%, 시간 가치 하락)[1][2].
4. **보고서 분석**: 기여 > 모델 비교(데이터 기반 vs 라스트 클릭). 예: Meta 경로 "Google→Meta→구매" → Meta30%[1][5].
5. **BigQuery 연동**: GA4 내보내기 → SQL 쿼리 `SUM(전환_가치 * 기여_비율) / 비용`으로 채널 ROAS[3].
6. **Looker Studio 대시**: GA4+Meta 데이터 소스 조인 → 크로스채널 대시(ROAS/MTA 지표)[4].
7. **최적화**: Google Ads 스마트 입찰 연동(데이터 기반 신호 실시간)[2]. A/B 테스트: 데이터 기반 vs 선형(동등33%)[1].

### LLM 기반 마케팅 인사이트 자동 생성 프롬프트 설계: 할루시네이션 방지 + 액션 가능한 출력 구조
# LLM 기반 마케팅 인사이트 자동 생성 프롬프트 설계

## 핵심 프레임워크

**LLM 마케팅 자동화의 본질은 '프롬프트 엔지니어링'과 '파이프라인 설계'**[2]입니다. 단순 질문이 아닌 **구조화된 프롬프트 템플릿 → API 연동 → 출력 후처리 → 액션 실행**까지의 시스템적 흐름을 구축해야 합니다[2][3].

## 할루시네이션 방지 전략

### 1. 데이터 기반 제약 조건 주입
프롬프트에 **검색량, 경쟁도, CTR 데이터**를 함께 제공하여 LLM이 실제 데이터를 근거로 생성하도록 유도합니다[2]. 단순 주관적 분석이 아닌 수치 기반 인사이트를 강제합니다.

### 2. 명확한 역할과 제약 정의
- **역할 부여**: "너는 뛰어난 마케팅 데이터 분석가야"[1]
- **금지 표현 및 브랜드 가이드라인을 시스템 프롬프트에 주입**[2]하여 일관된 문체와 신뢰도 유지

### 3. 구조화된 단계별 처리
```
1단계: 키워드/데이터 입력 → 검색 의도 분석
2단계: 콘텐츠 브리프 자동 생성 → 구조 및 소제목 기획
3단계: 초안 생성 → 브랜드 톤앤매너 적용
```
[2]

### Supermetrics, Whatagraph, Databox AI 인사이트 기능 역기획: 신호 서술 프레임워크

**AI 인사이트 기능은 마케팅 데이터(GA4, Meta Ads 등)에서 이상치/트렌드/상관관계를 자동 탐지해 '신호( signal )'를 자연어로 서술. 핵심은 '무엇(what)', '왜(why)', '영향(impact)' 구조로 압축 전달하며, 품질 저하 신호(예: 반복 패턴, 일관성 상실)를 장기 관찰로 검증[1].**

#### 1. **공통 신호 탐지 로직 (세 도구 합성)**
- **데이터 소스**: Supermetrics(100+ 커넥터, GA4/FB Ads 실시간), Whatagraph(시각화 중심), Databox(모바일 대시보드) 모두 동일[내 지식 보강: 공식 문서 기준].
- **신호 유형** (우선순위 순):
  1. **트렌드 신호**: "ROAS 25%↑ (지난주 대비), 트래픽 15%↓로 인한 효율화[Supermetrics AI]".
  2. **이상치 신호**: "클릭 300% 스파이크, A/B 테스트 승자 캠페인 영향[Databox AI]".
  3. **상관 신호**: "인스타그램 조회수↑ → 전환 12%↑, 콘텐츠 최적화 추천[Whatagraph]".
- **서술 템플릿**: **[메트릭 변화 수치] + [기간/비교] + [원인 가설] + [액션 제안]**. 예: "전환율 18%↓ (7D), 랜딩페이지 이탈↑ → A/B 테스트 실시".
