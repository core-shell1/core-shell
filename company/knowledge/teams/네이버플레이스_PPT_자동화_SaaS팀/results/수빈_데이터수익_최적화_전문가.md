## 🎯 측정 대상: SaaS 핵심 지표 정의서

### 1. 북극성 지표: MRR (Monthly Recurring Revenue)
```sql
-- 계산식 (PostgreSQL 쿼리)
SELECT 
  DATE_TRUNC('month', created_at) as month,
  SUM(CASE 
    WHEN plan_type = 'subscription' THEN amount 
    ELSE 0 
  END) as mrr,
  SUM(CASE 
    WHEN plan_type = 'pay_per_use' THEN amount 
    ELSE 0 
  END) as one_time_revenue
FROM payments
WHERE status = 'completed'
GROUP BY month
ORDER BY month DESC;
```

**목표값**:
- Week 4: ₩500,000 (구독 20명 × ₩25,000/월)
- Week 12: ₩2,000,000 (구독 80명)

**측정 주기**: 매일 오전 9시 자동 계산 → Slack 알림

### 2. 전환 퍼널 (5단계)

```
[가입] → [첫 PPT 생성] → [3회 사용] → [첫 결제] → [구독 전환]
 100%      60%           35%         20%        15%
```

**각 단계 정의 & 추적 이벤트**:

| 단계 | 이벤트명 | 측정 기준 | 목표 전환율 | 드롭오프 원인 분석 |
|------|---------|----------|------------|------------------|
| **가입** | `user_signup` | 이메일 인증 완료 | - | 인증 메일 미도착(스팸함) |
| **첫 생성** | `first_ppt_created` | 가입 후 24시간 내 생성 | 60% → 75% | URL 입력 실패, 크롤링 에러 |
| **3회 사용** | `third_ppt_created` | 첫 생성 후 7일 내 | 35% → 50% | 가치 체감 부족, 경쟁사 이탈 |
| **첫 결제** | `first_payment` | 무료 크레딧 소진 후 | 20% → 30% | 가격 저항, 결제 UX 마찰 |
| **구독 전환** | `subscription_started` | 건당 결제 3회 후 | 15% → 30% | 구독 혜택 인지 부족 |

**추적 코드 예시** (FastAPI):
```python

@router.post("/generate")
```

### 3. 이탈률 (Churn Rate)

**정의**: 당월 구독 취소 / 전월 총 구독자 × 100

```sql
-- 월별 이탈률 계산
WITH monthly_subs AS (
  SELECT 
    DATE_TRUNC('month', created_at) as month,
    user_id
  FROM subscriptions
  WHERE status = 'active'
),
churned AS (
  SELECT 
    DATE_TRUNC('month', canceled_at) as month,
    user_id
  FROM subscriptions
  WHERE status = 'canceled'
)
SELECT 
  m.month,
  COUNT(DISTINCT c.user_id)::FLOAT / NULLIF(COUNT(DISTINCT m.user_id), 0) * 100 as churn_rate
FROM monthly_subs m
LEFT JOIN churned c ON m.month = c.month
GROUP BY m.month;
```

**목표**: <5% (월간 기준)

**이탈 예측 시그널** (트리거 이메일 발송 조건):
| 시그널 | 정의 | 액션 |
|--------|------|------|
| 로그인 14일 미접속 | `last_login_at < NOW() - INTERVAL '14 days'` | "마지막 방문 이후 이런 기능이 추가됐어요" 이메일 |
| 사용량 50%↓ | `current_month_usage < prev_month_usage * 0.5` | "혹시 불편한 점이 있으신가요?" 설문 |
| 결제 실패 | `payment_status = 'failed'` | 24시간/72시간 자동 재시도 + SMS 알림 |

### 4. LTV (Lifetime Value) & CAC (Customer Acquisition Cost)

**LTV 계산**:
```
LTV = ARPU × (1 / Churn Rate) × Gross Margin
    = ₩25,000 × (1 / 0.05) × 0.7  
    = ₩350,000
```
- ARPU: 월 평균 구독료 ₩25,000
- Churn: 5% (월간)
- Gross Margin: 70% (AWS 비용 30% 차감)

**CAC 계산**:
```
CAC = 월 마케팅 비용 / 신규 고객 수
    = ₩600,000 / 30명  
    = ₩20,000
```

**목표 LTV:CAC = 17.5:1** (현재 기준, 업계 우수 수준)

**CAC 절감 전략**:
- 무료 채널 우선: 네이버 블로그 SEO, 소상공인 커뮤니티 참여
- 유료 광고는 MRR ₩1,000,000 이후 테스트 (네이버 검색광고 ₩100,000/월)

## 🔧 추적 시스템 구축 (4주 MVP)

### Phase 1: PostgreSQL 이벤트 테이블 (Week 1)

```sql
-- 이벤트 추적 테이블
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    user_id VARCHAR(50) NOT NULL,
    event_name VARCHAR(100) NOT NULL,
    properties JSONB,  -- 유연한 속성 저장
    created_at TIMESTAMP DEFAULT NOW(),
    
    INDEX idx_user_event (user_id, event_name),
    INDEX idx_created_at (created_at)
);

-- 예시 레코드
INSERT INTO events (user_id, event_name, properties) VALUES
('user_123', 'ppt_created', '{"url": "https://place.naver.com/...", "is_first": true}'),
('user_123', 'payment_completed', '{"amount": 5000, "plan": "pay_per_use"}');
```

**핵심 이벤트 15개** (우선순위순):
1. `user_signup` (가입)
2. `email_verified` (이메일 인증)
3. `first_ppt_created` (첫 생성)
4. `ppt_created` (일반 생성)
5. `ppt_download` (다운로드)
6. `payment_completed` (결제 완료)
7. `payment_failed` (결제 실패)
8. `subscription_started` (구독 시작)
9. `subscription_canceled` (구독 취소)
10. `feature_used` (특정 기능 사용, properties에 feature_name)
11. `error_occurred` (에러 발생, properties에 error_type)
12. `support_ticket` (문의 접수)
13. `login` (로그인)
14. `logout` (로그아웃)
15. `referral_sent` (추천 링크 발송)
