
### 인테리어 자재 공급업체 신용도 평가 데이터 소스 — 나이스평가정보, 기업신용평가 API, 공공데이터포털 사업자 정보 연계 방법
# 인테리어 자재 공급업체 신용도 평가 데이터 소스

## 주요 데이터 소스 3가지

**1. 기업신용평가 서비스 (KCR - KODATA)**

기업신용평가서는 3단계로 구분되며, 협력업체 신용도 평가에 실질적으로 활용됩니다[1]:

| 평가 등급 | 특징 | 활용 대상 |
|---------|------|---------|
| KCR-1 (예비) | 현장조사 없이 제출자료 기반 약식평가 | 협력업체 등록용 간이평가 |
| KCR-2 (표준) | 현장조사 + 재무상태 체계적 분석 | 대기업/중견기업 거래처 리스크관리 |
| KCR-3 (종합) | KCR-2 + 여신분석/매출처분석 추가 | 주요 거래처 관리 |

**2. 공공데이터포털 인테리어업체 DB**

지방자치단체 관리 인테리어업체(실내건축업) 표준데이터가 연계 가능합니다[2]. 제공 항목은 업체명, 소재지, 전화번호, 등록일자, 등록번호, 대표자명, 총직원수이며 연간 갱신주기로 운영되고 있습니다[2].

**3. NICE 개인신용평가시스템 (CSS)**

한국주택금융공사 용역계약 현황에 따르면 나이스에서 대안정보 용역을 제공 중입니다[4].

## 실무 연계 프레임워크

**데이터 통합 단계**:
- 공공데이터포털 사업자정보 → 기본정보(등록번호, 대표자) 추출
- KODATA KCR API → 기업신용평가 등급 및 재무정보 연동
- 나이스평가정보 → 대표자 신용도 보조 확인

**주의사항**: 공공데이터는 월초 병합으로 제공되므로 시점별 시차가 발생할 수 있습니다[2].

### 건축·인테리어 인허가 체크리스트 자동화 — 용도변경, 실내건축 허가 기준, 지역별 조례 차이 데이터 구조화 방법
### **핵심 데이터 구조화 프레임워크**
건축·인테리어 인허가 체크리스트 자동화는 **법령 구조화 DB + 조건별 룰 엔진 + 사용자 입력 매핑**으로 구현. 용도변경/실내건축은 **키워드 추출 → 법령 필터링 → 체크리스트 동적 생성** 워크플로우 적용[1][2][4].

#### **1. 데이터 구조화 방법 (법령 → DB 변환)**
- **법령 파싱 계층**: 
  | 계층 | 내용 | 예시 데이터 | 출처 |
  |------|------|-------------|------|
  | **상위** | 국가법령 (건축법, 시행령, 시행규칙) | 140개 법규 자동 분석[2] | [2][3] |
  | **중위** | 지자체 조례 (지역별 용도/면적 제한) | 서울 vs. 원주시 체크리스트 차이[4][9] | [4][9] |
  | **하위** | 세부 기준 (용도변경: 기존용도+신용도 비교, 실내건축: 면적 200㎡ 초과 시 허가) | 키워드 DB(용도, 면적, 높이)[1] | [1][7] |
- **구조화 스텝** (KR20220091761A 특허 기반, 8단계 워크플로우)[1]:
  1. 법령 텍스트 → **키워드 1차 도출** (e.g., "용도변경", "실내건축 허가면적").
  2. 키워드 DB 저장 + 사용자 수정.
  3. 프로젝트 입력 (토지 위치, 용도, 면적) → **전산 알고리즘 필터링** (조건 만족 법령 도출).
  4. 체크리스트 DB에서 **법령별 템플릿 취합** (Word/HWP 출력).
  5. 사용자 편집 → 계정별 DB 저장.
- **지역별 차이 처리**: 조례 DB 별도 구축 (e.g., 원주시 체크리스트: 신속허가과 참고용[9]). API 연동으로 100+ 지자체 실시간 로드[4].

#### **2. 용도변경 자동화 체크리스트**
- **입력 매핑 룰**: 기존용도(상가) → 신용도(주거) 시 **법률 비교 테이블 생성**.
  | 체크항목 | 기준 수치 | 자동 검토 로직 | 출처 |
  |----------|-----------|---------------|------|
  | 용도 호환성 | 화재/환기 기준 상향 | BIM 모델 매핑 (IFC 활용)[3][7] | [3][7] |
  | 면적 제한 | 1,000㎡ 초과 → 심의 | 토지필지+용도 입력 3초 검토[6] | [6] |
  | 구조 변경 | 철근 콘크리트 → 경량 | 3D 법규 검토 (하우빌드 스케치)[6] | [6] |
- **사례**: buildingcode.co.kr – 입력 후 140법규 자동 출력 + 지역조례 HWP 다운[2]. 오류율 5% 미만 (BIM 기반)[3].

#### **3. 실내건축 허가 기준 자동화**
- **허가 트리거**: 실내 면적 **200㎡ 초과 OR 구조 변경** 시 필수[1][2].
  | 기준 | 수치/조건 | 자동화 출력 | 출처 |
  |------|-----------|-------------|------|
  | 면적 | 200㎡~ | 체크리스트 + 별표 HWP | [2] |
  | 방화/피난 | 구획벽 R-60 이상 | BIM 2D/3D 검토[8] | [8] |
  | 환기 | 공기량 20㎥/인 | 키워드 룰 엔진[1] | [1] |
- **사례**: BIRA 적법성검토 – DB 구조화로 조건별 체크리스트 생성, 전국 확대[4]. AI 코파일럿: 10단계(허가~준공) 28 체크리스트[5].

### 지역별 인테리어 자재 유통 구조 분석 — 직거래 vs 대리점 마진 구조, 스펙 매칭 DB 설계 사례
### 한국 건설/인테리어 자재 유통 구조: 직거래 vs 대리점 마진

**직거래(직접구매)가 55~60% 주도, 대리점(간접) 40~45%로 마진 10~30% 차이 발생.** 대형 건설사/인테리어 플랫폼은 물류 축소로 **최대 70% 할인** 실현, 소규모는 대리점 의존[1][2][5].

#### 유통 구조 비중 및 마진 비교 (전문건설업 기준)
| 유통 방식 | 비중(%) | 마진 수준 | 적용 사례 | 장단점 |
|-----------|---------|-----------|-----------|--------|
| **직거래 (제조사 → 건설사/인테리어사 직접)** | 55~60 | 제조원가 +5~10% (물류/협상 비용만) | 대량 공공공사 (공사용자재 직접구매제도), LX하우시스 B2B[1][3][5] | **장점**: 가격 20~70%↓, 재고 리스크↓. **단점**: 소량/수시 구매 부적합. |
| **대리점 (제조사 → 도·소매 → 건설사)** | 40~45 | 제조원가 +20~30% (대리점 마진 10~15% + 도매 5~10%) | 소량 민간공사, 지역 철물/목재상[1][2] | **장점**: 즉시 공급. **단점**: 총 마진 누적, 자재비 50%↑ (인테리어 비용 50% 차지)[2]. |

- **시장 규모**: 94.6조원 (종합건설 53.1조, 전문 41.5조), 제조원가 33.8%[1].
- **지역 변동**: 공공공사 직거래 ↑, 민간 ↓. 당진(충남) 예: 원당철강/영일목재 등 지역 대리점 10곳+ 밀집, 직거래 대형 프로젝트 한정[4].

#### 직거래 최적화 사례: 인테리어브라더스 '달콤한 자재마켓'
- **모델**: 온라인 편집숍 + 물류창고 직판. 브랜드 숨김 → 가격 경쟁력 ↑, 재고 부담 ↓ 판매업체 유치[2].
- **성과**: 자재 **70% 할인**, 인테리어 비용 50% 절감. 소비자 예산/공간 입력 → 2분 매칭 (컨시어지)[2].
- **확장**: B2C 다각화 (LX하우시스 '20년 진입)[3].
