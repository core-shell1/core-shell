| 접근 방식 | 장점 | 단점 | 사용 시기 |

**법률 도메인 데이터 품질 검증은 의미 보존과 법적 일관성을 동시에 확보하는 3층 구조로 설계**되어야 합니다[1].

| 검증 계층 | 목표 | 실행 방식 |
|---------|------|---------|
| **의미 단위 분석** | 법령의 논리 구조 파악 | 원문을 의미 단위로 전처리하여 안전설계 Rule 도출[3] |
| **법적 일관성 검증** | 법령-조항 간 관계 명확화 | 관련 법령 원문을 검색·추가하여 해석 정보 보충[1] |
| **자동 검토 체계** | Rule Checker 기반 오류 탐지 | 2D 설계도면 등에서 기준 적용 여부를 모듈식으로 검증[3] |

## 1. 누락(Missing Data) 탐지 규칙

**법령정보 재지정 방식**: 원시 데이터에서 법령 해석 정보가 축약적이거나 구체적 사유가 요약된 경우, 검색 모델을 활용하여 관련 법령 원문 조항을 추가[1]. 이를 통해 거대언어모델의 법률적 해석 및 적용 능력을 향상시킬 수 있습니다.

**구체적 규칙**:
- 약관법 제조의 위법성 판단 근거에는 다양한 법령이 참고되므로, 해당 관련 법령 원문을 검색하여 원시데이터에 추가[1]
- 안전설계 기준 존재 유무를 1차적으로 판단할 때 대상 화학물질 인지 여부와 물리화학적 특성 확인 후 추가 기준 필요성 판별[3]
- 검토 누락 방지: Rule Checker를 모듈식으로 개발하여 각 기준을 개별적으로 검토되도록 함으로써 누락 가능성 감소[3]

---

## 2. 충돌(Conflict) 탐지 규칙

**법적 책임·의무·권리 구조 유지**: 프롬프트 기반 문장 증강 과정에서 의미보존형(semantic-preserving) 규칙을 적용하여 법적 책임의무권리 구조를 유지하면서 표현 다양성 확보[1]. 이는 서로 다른 법령 간 충돌을 사전에 방지합니다.

**구체적 규칙**:
