### Gemini 1.5 Pro vision vs GPT-4V for dense text extraction from images — accuracy benchmark and cost comparison
**Gemini 1.5 Pro vision outperforms GPT-4V (GPT-4o) in **dense text extraction accuracy** from receipts (factuality 91% vs 84%, Levenshtein 88% vs 85%) and uses 3.5x fewer input tokens, cutting costs ~70% ($7/M input vs $30/M).** GPT-4o edges out in broader vision benchmarks like DocVQA (93% vs implied lower for Gemini).[2][5][6]

### **Accuracy Benchmarks (Dense Text Extraction)**
Receipt extraction eval (image → key-value pairs, n=unspecified, Braintrust 2024):
| Model              | **Factuality** | **Levenshtein** | Notes |
|--------------------|----------------|-----------------|-------|
| **Gemini 1.5 Pro** | **91%**       | **88%**        | Top; 3% variance on reruns [2] |
| Gemini 1.5 Flash  | 88%           | 86%            | Faster (2.2s) [2] |
| **GPT-4o (4V)**   | **84%**       | **85%**        | 13s latency [2] |
| GPT-4o mini       | 81%           | 87%            | 38k tokens/image [2] |

- **Gemini wins on factuality** (LLM-judged via GPT-4o autorater); Levenshtein (edit distance) comparable.[2]
- **TV news OCR fail**: Both hallucinate heavily; GPT-4 > Gemini but neither production-ready vs classical OCR.[1]
- **DocVQA** (doc Q&A): GPT-4o 93%, Gemini 1.5 Pro ~81-89% range (chart-related).[5]
- **General vision**: GPT-4o leads (ChartQA 86%, MathVista 64%); no direct dense text focus.[5]

**실전 프레임워크**: 
1. Test 10-50 images (receipts/docs) on **factuality** (JSON output match) + **Levenshtein**.
2. Threshold: >90% factuality → deploy; else hybrid OCR + LLM.
3. Rerun evals (variance ±3%).[2]

| Model            | Input     | Output    | **Total Savings** |
|------------------|-----------|-----------|-------------------|

**Recommendation**: **Gemini 1.5 Pro** for **cost-accuracy** in production dense text (e.g., invoices); GPT-4o if DocVQA/reasoning critical. Always validate vs Tesseract/PaddleOCR.[1][2]

### How to handle multi-slide Instagram carousels in sequence for context-aware content extraction — chaining image analysis prompts
### **핵심 프레임워크: Sequential Carousel Chain Analysis (SCCA)**
Instagram 멀티 슬라이드 캐러셀(최대 20슬라이드[4])을 순차 처리해 **컨텍스트 인식 콘텐츠 추출**하는 최적 방법. **슬라이드 1이 80% 결정적**이며[2], 스와이프 신호로 알고리즘 부스트(1.4x 리치[1]). AI 프롬프트 체인으로 **스와이프율 1.92% 달성** 평균 대비 최상위[1].

#### **1. 초기화: Hook/Context Extraction (Slides 1-2)**
- **프롬프트 1 (Slide 1 단독 분석):** "이 이미지의 **핵심 훅(hook)** 식별: 질문/통계/문제/볼드 스테이트먼트 추출. 정보 갭(information gap) 요약 (e.g., '95%가 이 실수 함'[3]). 배경 컨텍스트 무시."
  - **목적:** 스크롤 스톱 여부 판단. **80% 무게[2]** – 실패 시 전체 스킵.
- **프롬프트 2 (Slide 1+2 체인):** "Slide 1 훅 [{prev_summary}]에 Slide 2 추가: 문제陳述(problem statement) 또는 '왜 중요한가' 추출. **PAS 구조** 적용 (Problem-Agitate[2]). 누적 컨텍스트: {hook} + {context}."
  - **예시 출력:** "Hook: '10k 손실 배운 교훈'. Context: 비즈니스 실패 원인. 누적: PAS Setup."

**성능 수치:** Slide 1 스와이프율 1차 지표 – Insights에서 확인 (Instagram 2026 업데이트[1]).

#### **2. 본문 체인: Value Delivery Chain (Slides 3-10, 8-10슬라이드 최적[1])**
- **반복 프롬프트 체인 (각 슬라이드 N):** "이전 누적 컨텍스트 [{chain_summary}]에 Slide {N} 추가. **하나의 액션 팁/인사이트**만 추출 (1-2초 읽기[1]). bullet/번호/차트 형식 유지. 전환어(transition) 연결: '다음으로', '이제 해결'."
  - **체인 로직:** JSON 배열 누적 `{ "slide1": "...", "slide2": "...", "cumulative_narrative": "전체 스토리 아크" }`.
  - **프레임워크 적용:**
    | 구조 | 슬라이드 | 프롬프트 포커스 | 예시 케이스 |
    |------|----------|-----------------|-------------|
    | Setup[3] | 1-2 | Hook + Problem | "통계 충격 → 왜 안됨?" |
    | Development[3] | 3-8 | Tension Build (장애/인사이트) | 튜토리얼 스텝[2], 데이터 브레이크다운 |
    | Resolution[3] | 9-10 | Solution + CTA | "이렇게 해라 → 링크" |
  - **최적화:** 텍스트 2-3문장/슬라이드, 1080x1350px[1]. 시각 흐름 일관(컬러/폰트[2]).

**실전 사례:** PostNitro AI 워크플로[3] – 소스 분석 → 훅 생성 → 슬라이드 전환 자동화. 1시간 → 분 단위 변환.

#### **3. 종료: Resolution/CTA Extraction + Validation (Final Slides)**
- **프롬프트 3 (Last 2 Slides):** "전체 체인 [{full_chain}]에 최종 슬라이드 추가. **솔루션/결과/CTA** 추출 (e.g., '구매 링크'). Completion rate 예측: 논리 흐름 점검."
- **검증 프롬프트:** "전체 추출 검토: **스토리 아크 완성도** (Setup→Dev→Res[3])? 누락 컨텍스트? 메타데이터(브랜드/테마[4]) 추가."
