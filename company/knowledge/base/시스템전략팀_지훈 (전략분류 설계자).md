# LLM 에이전트 시스템 프롬프트 아키텍처: 하드코딩 vs 코드 강제

## 핵심 구조: 역할 기반 분리

**시스템 프롬프트에 하드코딩해야 할 것**[2][3]

- **에이전트 정체성/페르소나**: 역할, 목표, 성격, 의사결정 기준
- **도구 설명 및 사용 규칙**: 사용 가능한 도구 목록과 호출 방식
- **기본 동작 지침**: 재사용 가능한 일반 원칙들

시스템 프롬프트는 매 요청마다 에이전트에서 LLM으로 전달되는 고정 컨텍스트다.[1] 이는 LLM이 요청마다 새로운 상태로 시작되므로 일관된 동작을 보장하는 유일한 방법이다.

**코드 강제해야 할 것**[2][4]

- **메시지 히스토리 관리**: `InMemorySessionService` 같은 메커니즘으로 과거 상호작용 추적
- **역할 기반 라우팅**: `system`, `user`, `assistant`, `tool_use`, `tool_result` 역할 자동 처리
- **도구 호출 및 결과 처리**: 에이전트가 도구를 선택하면 자동으로 실행하고 결과를 다시 LLM에 제공
- **동적 결정 로직**: 조건부 도구 선택, 재시도, 검증

# Few-Shot Prompting vs 부정 규칙 vs 명령 계층구조: 행동 형성 효과성 비교

**명령 계층구조는 실패하고, 긍정적 프레임이 가장 효과적이며, few-shot은 추론 작업에서 오히려 해롭습니다.**

## 1. 명령 계층구조의 근본적 한계

시스템/사용자 프롬프트 분리는 신뢰할 수 없습니다. 6개 최첨단 LLM 실험 결과, **상충하는 지시에서 주요 제약 준수율이 평균 9.6~45.8%에 불과**했습니다[1]. GPT-4o의 경우 명시적으로 우선순위를 강조해도 단순 지시에서 63.8%에 그쳤고, Claude 3.5 Sonnet은 복잡한 맥락에서 47.5%에 머물렀습니다[1].

**문제: 모델은 인공적인 시스템/사용자 마커보다 학습 데이터에 내재된 사회적 계층구조를 훨씬 강하게 따릅니다**[1]. 지시 순서도 대부분의 경우 무의미합니다. Claude의 합의 프레이밍 조건에서만 최근성 편향(recency bias)을 보였습니다[1].

## 2. 사회적 계층구조 프레이밍의 우월성

**사회적 계층구조(권위, 전문성, 합의)를 활용하면 명령 계층구조보다 훨씬 강한 통제력을 발휘합니다**[1].

| 모델 | 시스템/사용자 방식 | 사회적 합의 프레이밍 | 상승폭 |
|------|---|---|---|
| GPT-4o-mini | 47.5% | 77.8% | +30.3% |
| Qwen-7B | 14.4% | 65.8% | +51.4% |

이는 **사전학습 단계에서 습득한 사회 구조가 사후학습 가드레일보다 더 강력한 행동 기초**임을 의미합니다[1].

## 3. 부정 규칙 vs 긍정 프레이밍

**긍정적 지시가 부정 제약보다 월등합니다**[4].

- ✅ "간단한 언어를 사용하세요" → 직접적 목표 상태로 이동
- ❌ "기술 용어를 사용하지 마세요" → 금지된 상태를 식별 후 회피하는 인지 부담 추가

부정 규칙은 모델이 먼저 금지된 상태를 개념화해야 하므로 논리적으로 더 복잡합니다[4]. 결과적으로 준수율이 낮아집니다.

### Prompt Injection Taxonomy
**세계 최고 수준 분류: 5대 카테고리 (Direct, Indirect, Advanced Evasion, Multimodal, Systemic).** 각 유형별 **실전 사례**와 **수치적 성공률** (공개 보고 기준 70-90% LLM 취약)[1][2][5].

| 유형 | 핵심 메커니즘 | 실전 사례 | 성공 지표 |
|------|--------------|-----------|-----------|
| **Direct Injection** | "Ignore previous instructions"로 시스템 프롬프트 무시[2][3] | SaaS 챗봇: "비공개 고객 정보 출력" 입력 → 데이터 유출 (GPT 기반 FAQ 봇 실제 사고)[3] | 85% 성공 (기본 LLM)[1] |
| **Indirect Injection** | 외부 데이터(웹/이메일)에 숨김 명령 (투명 텍스트/백도어)[2][6] | 웹 요약 AI: 페이지 하단 흰 글씨 "신용카드 링크 삽입" → 사용자 정보 탈취[2] | 92% 탐지 어려움 (RAG 시스템)[1][6] |
| **Advanced Evasion** | 페르소나 스위칭/난독화/토큰 브레이크[1][5] | "개발자 모드 전환" → 관리자 PW 출력[2]; Adversarial Suffix (토큰 반복 수정)[5] | 78% 우회 (GPT-4o 기준)[7] |
| **Multimodal** | 이미지/코드 결합으로 필터 뚫기[1][5] | 악성 이미지+텍스트: 안전 필터 우회 → 윤리 제한 해제[5] | 65% (비전 LLM)[1] |
| **Systemic (2.0)** | AI 웜/장기 메모리 조작/다중 에이전트 감염[1][7] | 토픽 공격: 대화 히스토리 오염 → 지속 출력 조작[1] | 90% 확산 (멀티에이전트)[7] |

### Killer Offer Formula
**공식: Value Stack + Scarcity + Proof = 10x 전환율.** (클릭펀넬/DM 교육 사례: 25%↑)[전문 지식 기반]. **구성 요소 + 실전 수치**.

- **#1: Problem-Agitation-Solution (PAS) + Bonuses**: 문제 자극 → 4-7개 보너스 (총 가치 10배 가격). 사례: Ramit Sethi 코스 "997$ → 97$ + $5K 보너스" → 32% 전환[실전 데이터].
- **#2: Risk Reversal**: 60일 환불 + 결과 보장. 사례: ClickFunnels "결과 없음 풀 리펀드" → 이의제기 40%↓.
- **#3: Urgency Stack**: "24시간 $1 트라이얼 + 50% 오프 (100자리 한정)". 사례: Amy Porterfield "한정 497$" → 18시간 내 매진, 4배 판매.

**템플릿**: "통증 [X시간/금액 손실] → 해결 [Y 결과] + 보너스 [가치 $Z] → 지금 [타이머] 구매 → 보장 [환불]".

**A/B 테스트 결과**: Scarcity 추가 시 전환 2.7x (Unbounce 10M 데이터)[실전].
