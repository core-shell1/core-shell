### **성능 비교 (2024 기준, 모바일/데스크톱 기준)**
| Technique | FPS (60fps 기준, iPhone 14/Chrome) | CPU 사용량 | 장점 | 단점 | 추천 용도 |
|-----------|------------------------------------|------------|------|------|----------|
| **SVG feTurbulence** | 58-60fps[6][7][9] | 낮음 (GPU 가속) | 무한 스케일, 애니메이션 부드러움, 파일 0KB | 노이즈 패턴 고정적 (baseFrequency 조정 필요) | 대형 배경/전체 페이지 (99% 케이스) |
| **Canvas Noise** | 45-55fps (WebGL 사용 시 58fps) | 높음 (JS 렌더링) | 동적/랜덤 생성 가능, 인터랙티브 | JS 오버헤드, 배터리 소모 ↑ | 인터랙티브 요소만 (버튼/호버) |

**결론: SVG feTurbulence가 2024 최고 성능/효율.** Canvas는 JS 피하세요. PNG는 100x100px 이하로 압축[3][4].

### **SVG feTurbulence (최고 추천, GPU 최적화)**
**실전 코드 (전체 화면 오버레이):**
```html
<svg style="position:fixed;top:0;left:0;width:100%;height:100%;pointer-events:none;opacity:0.1;z-index:9999">
  <filter id="grain">
    <feTurbulence type="fractalNoise" baseFrequency="0.9713" numOctaves="4" result="noise"/>
    <feDisplacementMap in="SourceGraphic" in2="noise" scale="150" xChannelSelector="R"/>
  </filter>
  <rect width="100%" height="100%" filter="url(#grain)"/>
</svg>
```
- **파라미터 튜닝:** `baseFrequency: 0.7-1.0` (세밀도), `scale: 50-200` (강도), `numOctaves: 2-4` (층위, 4=고품질)[6][7][9].
- **성능 팁:** `color-interpolation-filters='sRGB'` 추가로 색상 정확도 ↑. 애니메이션 시 `turbulence` seed 변경 (JS minimal)[9].
- **테스트 사례:** 4K 화면 60fps 유지, 모바일 Safari/Chrome 100% 호환[7].

### **Canvas Noise (동적 필요 시 한정)**
**실전 코드 (WebGL 최소 JS):**
```js
const canvas = document.createElement('canvas');
canvas.style.cssText = 'position:fixed;top:0;left:0;width:100%;height:100%;pointer-events:none;opacity:0.1;z-index:9999';
document.body.append(canvas);
const ctx = canvas.getContext('2d');
canvas.width = innerWidth; canvas.height = innerHeight;
const imageData = ctx.createImageData(canvas.width, canvas.height);
for(let i=0; i<imageData.data.length; i+=4) {
  const noise = (Math.sin(i*0.01) + Math.sin(i*0.02)) * 128;
  imageData.data[i+3] = noise; // 알파만 노이즈
}
ctx.putImageData(imageData,0,0);
```
- **성능 팁:** requestAnimationFrame으로 30fps 제한, WebGL2 전환 시 SVG급[3]. 배터리 20% ↑ 주의.
- **용도:** 마우스 따라가는 노이즈만.

### 2. Magnetic Effect 핵심 수식 (버텍스 기반)
```javascript
// 요소 중심으로부터 마우스 거리 계산 → 비례 이동
function magneticEffect(target, e) {
  const rect = target.getBoundingClientRect();
  const centerX = rect.left + rect.width / 2;
  const centerY = rect.top + rect.height / 2;
  
  const distanceX = (e.clientX - centerX) * 0.3; // 30% 강도
  const distanceY = (e.clientY - centerY) * 0.3;
  
  gsap.to(target, {
    x: distanceX,
    y: distanceY,
    duration: 0.4,
    ease: "back.out(1.7)"
  });
}

// 마우스 엔터/리브 핸들러
targets.forEach(target => {
  target.addEventListener('mouseenter', (e) => magneticEffect(target, e));
  target.addEventListener('mouseleave', () => 
    gsap.to(target, { x: 0, y: 0, duration: 0.5 })
  );
});
```
**최적값**: `movement: 0.25~0.35`, `duration: 0.3~0.5s`, `ease: back.out(1.7)`[3][5].

### 3. Edge-only Magnetism (대형 요소 최적화)
```javascript
// 요소 경계 내에서만 magnetic 적용
function edgeMagnetic(target, e) {
  const rect = target.getBoundingClientRect();
  const { clientX, clientY } = e;
  
  // 20% 경계 안에서만 활성화
  if (clientX > rect.left + rect.width * 0.1 && 
      clientX < rect.right - rect.width * 0.1 &&
      clientY > rect.top + rect.height * 0.1 && 
      clientY < rect.bottom - rect.height * 0.1) {
    return; // 중앙에서는 무시
  }
  
  magneticEffect(target, e);
}
```
**CPU 절약**: 70% 영역 스킵 → 40% 성능 향상[3].

### Font Pairing 핵심
**Inter (body/UI)** + **Space Grotesk (headings/display)**: Inter의 중립적 정밀성(높은 x-height, 화면 최적화)과 Space Grotesk의 모노-영감 quirk(각진 터미널, 기술적 개성)가 대비. 랜딩페이지에서 헤딩 강조 + 본문 가독성 극대화[1][4][7].

- **실제 사례**: 개발자 툴/게임/테크 랜딩(예: Versus Hotels, AI/생태 프로젝트)[5][6].
- **권장 웨이트**: Space Grotesk **Medium(500)/Bold(700)** 헤딩, Inter **Regular(400)** 본문, **Medium(500)** 코드[7].

### CSS Grid complex layout patterns for high-end agency landing pages: asymmetric, overlapping, viewport units
# CSS Grid 고급 레이아웃 패턴: 에이전시 랜딩페이지 구현

## 핵심 프레임워크

**비대칭 레이아웃 (Asymmetric Grid)**[1]
- `grid-template-columns: 1fr 3fr 1fr` 또는 `2fr 1fr 1fr` 등 불균등 비율 조합
- `grid-template-areas`로 시각적 영역 정의 → 복잡한 구조를 직관적으로 구성
- 예: 헤더/네비/사이드바/컨텐츠/푸터의 계층적 배치[1]

**겹침 효과 (Overlapping Elements)**[3]
- `grid-column: 1 / span 2` + `grid-row: 2 / 4` 조합으로 셀 영역 중첩
- `z-index` 속성으로 레이어 순서 제어 (숫자 높을수록 앞배치)[3]
- 그리드 라인 번호(`grid-column: 3/4; grid-row: 1/2`) 정밀 제어[3]

**뷰포트 단위 통합**[1]
```css
.container {
  display: grid;
  grid-template-columns: repeat(12, 1fr);
  grid-template-rows: auto 1fr auto;
  min-height: 100vh; /* 뷰포트 기준 높이 */
  gap: 20px;
}
```
