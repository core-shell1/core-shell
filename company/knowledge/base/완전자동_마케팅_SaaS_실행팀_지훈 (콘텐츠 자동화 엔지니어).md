**스마트스토어 → 콘텐츠 생성 → Instagram Graph API 자동 포스팅** 흐름. n8n 또는 Python으로 1시간 내 구현 가능. 스마트스토어 판매 데이터(상품/주문)를 트리거로 이미지/캡션 생성 후 IG 릴스/피드 배포[1][2][6].

```
스마트스토어 API (판매 알림) 
↓ (Webhook/크론)

↓

↓

**성공 사례**: A사 스마트스토어 API 대시보드 + IG 자동화로 시간대별 판매 20%↑[8]. Meta 공식 API로 계정 영구 블록 0%[7].

  1. Instagram → 비즈니스 계정 전환.
  2. Facebook 페이지 생성 → IG 연동.
  3. Meta 개발자 센터: 앱 생성 → Instagram Graph API 권한 추가 (instagram_basic, instagram_content_publish).
  4. Access Token 발급: Graph API Explorer → 장기 토큰 연장 (60일 무한 갱신)[1][2].
     - Account ID 조회: `/me/accounts` 엔드포인트.
- **스마트스토어 API 준비**:
  - 네이버 개발자 센터: 앱 등록 → Seller API 권한 (판매/상품 조회).
  - Webhook 설정: 신규 주문/판매 시 트리거 (e.g., 상품명/이미지 Pull)[8].
- **툴 선택**:
  | 툴 | 장점 | 단점 | 추천 시나리오 |

  | **n8n** | 노코드, 무료, 템플릿[1][6] | 서버 필요 | 초보/빠른 테스트 |

### **2. 구현 단계 (n8n 버전, 20분)**
n8n 설치 (Docker: `docker run -p 5678:5678 n8n`). Facebook Graph API Credential: 장기 토큰 입력[1].

1. **트리거 노드**: Schedule (매일 10AM) 또는 Webhook (스마트스토어 판매 알림).
2. **스마트스토어 데이터 Pull**:
   - HTTP Request: `GET /v1/sellers/{sellerId}/orders` (최근 판매 Top3 상품 ID/이미지/이름 추출)[8].
3. **콘텐츠 생성**:
   - OpenAI 노드: 캡션 생성 (`"스마트스토어 히트 상품 {상품명}, 가격 {가격}. #쇼핑 #스마트스토어"`).
   - AI 이미지 (DALL-E): `{상품명} 제품 사진, 인스타 릴스 스타일`.
4. **IG 포스팅 (2단계)**[1][2]:
   - **노드1: Container 생성** (Facebook Graph API 노드, v23.0):
     ```
     Resource: instagram
     Operation: Create Media Container
     Account ID: {IG 계정 ID}
     Query: image_url={AI 이미지 URL}, caption={캡션}
     ```
     → Container ID 반환 (5-10초 대기).
   - **노드2: Publish**:
     ```
     Resource: instagram
     Operation: Publish Media
     Account ID: {IG 계정 ID}
     Creation ID: {{이전 노드 Container ID}}
     ```
     → 즉시 업로드 (릴스: media_type=REELS).
5. **예약/반복**: Cron 노드 + IF (판매량>10 → 포스팅).

**테스트 지표**: 1회 실행 → 95% 성공률 (토큰 유효 시)[1].

# 콘텐츠 자동화 SaaS의 브랜드 일관성 유지 실전 전략

**SaaS 환경에서 AI 콘텐츠 자동화는 "사람-AI 하이브리드 워크플로우 + 템플릿화된 프롬프트 구조"로 일관성을 확보**합니다.

## 핵심 실행 프레임워크

### 1. 자동화 워크플로우 설계 (기획 → 검수)

체계적인 프로세스가 품질 편차를 제거합니다.[2][4]

**프로세스 흐름:**
- 기획 → 초안 생성 → 검수 → 게시 → 업데이트

이 구조를 문서화하면 **콘텐츠 품질과 일관성이 자동으로 유지**됩니다.[2] Zapier나 Make 같은 워크플로우 자동화 도구로 FAQ Schema나 HowTo 블록이 일정 주기로 자동 업데이트되도록 설정할 수 있습니다.[4]

### 2. 개인화된 프롬프트 템플릿 구조

**"사용자 질문 + 의도 + 브랜드 맥락 + 출력 형식"을 동시에 입력하는 프롬프트 구조를 템플릿화**하면, AI 생성 콘텐츠의 편차가 줄어듭니다.[4] 이는 단순 반복 생산이 아닌 **"브랜드 문법으로 자동화"하는 핵심**입니다.[4]

**실행 예시:**
- 변수화: `[산업용어] + [타겟오디언스] + [브랜드톤] + [형식(블로그/이메일/SNS)]`
- 결과: 동일 주제도 채널별·톤앤매너별로 일관되게 생성

### 3. 톤앤매너(Brand Voice) 명시화

**브랜드 보이스를 3~5개의 형용사로 정의**하고 템플릿에 포함시킵니다.[5]

**예시:**
- 전문적인 + 신뢰도 높은 + 혁신적인 + 친근한

여러 팀원이 콘텐츠를 올리더라도 이 기준이 일관성을 보장합니다.[5]

### 4. 사람의 개입 지점 (검수·브랜드 조정)

**핵심 7단계 플로우 (Time-to-Value 52% 단축, Activation 35-50% ↑):**[1][5]
1. **Frictionless Signup** (1분 이내): 이메일/소셜 로그인만 요구, 게스트 모드 허용.[2][4]
2. **Goal Segmentation** (2-3 질문): 역할(마케터/창업자), 목표(리드 생성/분석), 회사 규모(Solo/2-10/11-50).[4][5]
3. **Clear Value Prop + Checklist** (첫 화면): "40% 유저보다 앞서세요" 소셜 프루프 + 체크리스트(프로필 완성/첫 캠페인).[1][3]
4. **Interactive Tour** (3-5분): 툴팁/모달로 핵심 기능 핸즈온, "Aha!" 모멘트 유도.[1][6]
5. **Quick Win Trigger** (Day 1): 첫 성공(예: 이메일 발송) 후 보상(디지털 배지) + 다음 기능 unlock.[3][5]
6. **Progressive Profiling** (Day 1-3): 행동 기반 추가 질문, 맞춤 템플릿 제공.[1][4]
7. **Check-in + Feedback** (Day 7/30): 자동 이메일/채팅, 지원 센터 링크.[3][6]
