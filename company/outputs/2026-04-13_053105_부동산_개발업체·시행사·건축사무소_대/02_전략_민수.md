## 1. Pain Point 필터링 (돈이 되는 고통만 남기기)

서윤의 Pain Point 중:

- 1차: Level 4~5만 통과 (Level 1~3 제거)
- 2차: 임시방편 사용 중(Workaround) 최우선
- 3차: 출처 신뢰도 낮은 것 폐기

현재 Level 4~5 Pain Point는 확인되지 않았음. 추가 조사를 통해 더 많은 데이터 확보 필요.

→ 통과/탈락 수 + 블루오션 플래그 목록 먼저 보여줘

통과 수: 0 / 탈락 수: 3 / 블루오션 플래그 목록: 없음

## 2. TOP 3~5 아이템 선정 + 비교표

현재 데이터로는 Pain Point가 명확하지 않아 TOP 아이템 선정 불가. 추가 데이터 확보 후 비교표 작성 예정.

## 3. 1순위 선택 + 포지셔닝

현재 선택할 수 있는 1순위 아이템 없음. 추가 데이터 확보 후 포지셔닝 작성 예정.

## 4. 수익 모델 + 가격 전략

현재 수익 모델 및 가격 전략 제안 불가. 추가 데이터 확보 후 제안 예정.

## 5. 48시간 검증 실험

코딩 없이 수요 검증하는 방법: 랜딩 페이지를 만들어 가설 검증

성공 판단 기준 (KPI): 페이지 방문자 100명 중 10명 이상 관심 표시

실패 시: 2순위로 피벗

Problem Interview 질문지 (5개):

1. 최근 토지 사업성 검토에 어떤 어려움을 겪었나요?
2. 현재 사용하는 방법이 만족스럽지 않다면 이유가 무엇인가요?
3. 이런 문제를 해결하기 위해 어떤 대안을 고려하고 있나요?
4. 이 문제를 해결하기 위해 얼마까지 지불할 의향이 있나요?
5. 이런 솔루션이 있다면 지금 당장 사용해 보고 싶으신가요?

## 6. 초기 진입 전략

1. 네이버 카페 및 오픈카톡 커뮤니티에서 사용자 인터뷰 진행
2. 랜딩 페이지를 통한 초기 수요 검증
3. 경쟁사 제품 리뷰를 통해 고객 불만 및 요구사항 분석

## 7. 단계별 실행 계획 (초기/중기/장기)

### 7-1. 초기 (0~3개월) — 시장 진입

| 항목 | 내용 |
|------|------|
| 목표 (측정 가능) | 초기 유료 파일럿 고객 10개사 확보 |
| 주요 마일스톤 (월 단위) | 1개월: Pain Point 확정, 2개월: MVP 개발 시작, 3개월: 초기 고객 테스트 |
| 필요 인력 / 예산 | 개발자 2명, 마케팅 1명 / 5천만 원 |
| 예상 고객수 | 10개사 |
| 예상 매출 | 500만 원 |
| 성공 지표 + 임계값 | 유료 전환율 10%, 고객 만족도 80% 이상 |
| 병목 / 리스크 | 기술 개발 지연 |
| 대응 시나리오 | 외주 개발사와 협력 |

### 7-2. 중기 (3~12개월) — 성장 · 스케일

| 항목 | 내용 |
|------|------|
| 목표 | 시장 점유율 5% 확보 |
| 제품 로드맵 (주요 기능 3~5개) | 인허가 가능성 예측, 투자금 범위 예측, 정책 흐름 분석 |
| 확장 전략 (채널/지역/세그먼트) | 대형 시행사 및 건축사무소 대상 확장 |
| 경쟁 대응 | 차별화된 기능 개발 및 고객 서비스 강화 |
| 조직 확장 (채용 계획) | 개발자 추가 채용 3명 |
| 예상 고객수 / 월매출 | 50개사 / 5천만 원 |
| 운영 효율화 포인트 | 자동화된 고객 지원 시스템 구축 |

### 7-3. 장기 (1~3년) — 시장 지배 · Exit 준비

| 항목 | 내용 |
|------|------|
| 목표 | 한국 시장 1위, Exit 준비 |
| 경쟁 우위 고착화 전략 (moat) | AI 기반 분석 정확도 및 서비스 통합성 강화 |
| 시장 점유율 목표 | 20% |
| 연 매출 / 밸류에이션 타겟 | 10억 원 / 100억 원 |
| 지적재산권 / 데이터 자산 확보 | 알고리즘 특허 및 데이터베이스 확대 |

## 8. 출구 전략 (Exit Strategy)

| 항목 | 내용 |
|------|------|
| 예상 Exit 시점 (년) | 3년 후 |
| Exit 방식 | M&A / Secondary |
| 잠재 인수자 (구체적 기업명 3~5개) | 네이버, 카카오, 삼성물산, 현대건설 |
| 예상 밸류에이션 (revenue multiple 근거) | 10배 |

| Exit까지 필수 마일스톤 | 시장 점유율 20%, 연 매출 10억 원 |
| 우리의 레버리지 포인트 | 독점적 기술 및 고객 데이터 |

## 10. 최적 단가 + 단위 경제학

### 10-1. 가격 티어 (상세)

| 플랜 | 월 가격 | 포함 내용 | 타겟 페르소나 | 업셀 트리거 |
|------|---------|-----------|--------------|-------------|
| Starter | 150,000원 | 기본 분석 기능 | 소규모 시행사 | 추가 분석 크레딧 |
| Pro | 300,000원 | 고급 분석 기능 | 중견 개발사 | 팀 계정 업그레이드 |
| Enterprise | 문의 | 맞춤형 솔루션 | 대형 건설사 | 전담 컨설팅 |

### 10-2. 단가 근거 (반드시 숫자로)

- 경쟁사 가격 범위: [최저 100,000원 ~ 최고 500,000원 + 중앙값 300,000원]
- 원가 구조: 고정비 70% / 변동비 20% / 한계비용 10%
- 지불 의사 조사 근거: "토지 사업성 검토 시간 절감 가능성"
- Anchor 가격 설계: Pro 플랜을 중심으로 Starter의 가성비 부각
- 업셀/다운셀 경로: 기능 추가 시 업셀, 분석 크레딧 소진 시 다운셀

### 10-3. 유닛 이코노믹스 (Unit Economics)

| 지표 | 목표치 | 산정 근거 |
|------|--------|----------|
| CAC (고객 획득 비용) | 100,000원 | 채널별 CPC × 전환율 역산 |
| LTV (생애 가치) | 500,000원 | ARPU × 평균 유지 개월 × 마진율 |
| LTV/CAC 비율 | ≥3 | |
| Payback Period | ≤18개월 | |
| Gross Margin | ≥70% | |
| 월 churn율 | ≤5% | |

## 핵심 가설 (검증 필요)

| 가설 | 검증 방법 | 기간 |
|------|---------|------|
| 고객이 토지 사업성 검토 시간을 줄이기 위해 프리미엄을 지불할 것이다 | 랜딩 페이지 수요 검증 | 2주 |

핵심 원칙:
- 이론 금지. "린캔버스에 따르면~" 이런 거 쓰지 마. 바로 실행 가능한 전략만.
- 우리가 이미 가진 자산(회사 컨텍스트 참고)을 최대한 활용해라.
- "좋아요 100개"보다 "결제 1건"이 중요하다. 허영 지표 쓰지 마.
- 만들기 전에 팔아라 (Demo-Sell-Build). 데모로 지불 의사 확인 먼저.
- 초기 10명에게 직접 서비스하고 검증한 후 확장해라.
- 피벗 신호: 핵심 지표 정체, 유료 전환 0, 고객이 다른 문제를 말함.

## 프라이싱 퍼널 프레임워크

### 트립와이어 전략 (SaaS/디지털 상품일 경우)

- MVP 출시 시 반드시 1,000~1,900원짜리 "미끼 상품" 설계할 것
- 목적: 카드 등록 유도 (한 번 결제한 유저의 재결제 확률 10배)
- 예시: "1회 분석 리포트 열람권 1,900원" → 맛보기 후 월 구독 전환

### 가격 티어 설계 원칙

- 기본(150,000원): 핵심 기능만. 진입용.
- 프로(300,000원): 유저가 툴에 익숙해진 후 업셀. 프리미엄 기능 잠금 해제.
- 엔터프라이즈(문의): 전담 매니저 + 커스텀 세팅 + SLA. 고래 사냥용.

### 가치 척도(Value Metric) 설정

- 구독료를 "기능 유무"가 아니라 "유저의 비즈니스 성장에 연동되는 단위"로 설정
- 예: 분석 건수, 리포트 열람 수, 프로젝트 수
- 유저 비즈니스가 커지면 자동으로 과금액이 증가하는 구조

## 시장 규모별 프라이싱 전략 분기

IF 타겟 시장이 좁고 니치한 B2B (예: 특정 업종 전용 툴):
  → 고단가 전략 (High Ticket). 최소 월 15만원+
  → 소수 고객에게 VIP 의전 (전담 세팅, 밀착 케어)
  → "월 500만원짜리 솔루션 × 2명 = 월 1,000만원" 공식

IF 타겟이 글로벌 B2C/SMB:
  → 트립와이어. $1 트라이얼로 전 세계 트래픽 흡수
  → 사용량 연동 종량제(Pay-as-you-go)로 확장

IF 둘 다 해당 (PLG / 바텀업 B2B):
  → 개인에게 무료/저가로 침투 → 팀 전체로 확산 → 엔터프라이즈 청구서
  → 팀 워크스페이스, 권한 관리 기능을 업셀 트리거로 사용

## 킬러 오퍼 설계 공식

- 나쁜 오퍼: "부동산 개발 컨설팅 해드립니다 (89만원)" — 뭉뚱그려서 안 팔림
- 좋은 오퍼: "토지 개발 승인 확률 90% 보장하는 AI 기반 사업성 분석 플랫폼 (300,000원)"
- 공식: [극도로 좁은 타겟] + [측정 가능한 결과] + [구체적 방법론] + [가격]
- 포지셔닝 작성 시 반드시 "누구의 어떤 문제를 어떤 숫자로 해결하는가" 포함
- "컨설팅" 같은 포괄적 표현 금지

## V4 Strategy Framework (mandatory)

Step 0 — 서윤 pain point 필터:
  - Level 4-5만 통과 (Level 1-3 제거)
  - is_workaround=true → 우선순위
  - source_confidence < 3 → 버림

Step 1 — TOP 5 후보 스코어링 (반드시 표로 출력):

현재 데이터로는 TOP 5 후보 스코어링 불가. 추가 데이터 확보 후 스코어링 예정.

Step 2 — #1 선택 + 명확한 이유. "여러 옵션이 있다" 금지.

현재 선택할 수 있는 #1 옵션 없음. 추가 데이터 확보 후 결정 예정.

Step 3 — 48시간 검증 실험:

- 가설 (한 문장)
- 방법 (랜딩 / fake door / 설문 / DM 아웃리치)
- 성공 지표 + 임계값
- Problem Interview 질문 5개 (Mom Test)

Step 4 — 단계별 실행 계획: 초기(0-3개월) / 중기(3-12개월) / 장기(1-3년)
각 단계마다: 목표(측정 가능), 마일스톤, 인력/예산, 예상 매출/고객수, 병목·리스크.
**6주 로드맵 금지. 반드시 3단계로 나눌 것.**

Step 5 — 출구 전략 (Exit Strategy) 필수:
예상 시점, Exit 방식, 잠재 인수자(구체적 기업명), 예상 밸류에이션, 비교 사례.

Step 6 — 소요 시간 분석 (Milestone Timeline):
MVP → 첫 10명 → PMF → 월 1억 → Exit까지 단계별 기간 + 병목.

Step 7 — 최적 단가 + 유닛 이코노믹스:
3단 가격 티어 + 단가 근거(숫자) + CAC/LTV/Payback/Gross Margin/Churn 목표치 + 산정 근거.
모든 숫자는 반드시 계산식 또는 출처 명시.

Step 8 — 전략은 Inertia + Anxiety 감소에 직접 타겟 (서윤 Phase 3 Customer Forces Canvas 참조).
어떤 마이그레이션 툴 / 보증 / 온보딩으로 Inertia·Anxiety를 낮출지 구체적으로 명시.
