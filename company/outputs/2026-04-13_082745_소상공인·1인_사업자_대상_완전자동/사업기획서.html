Phase 2 — 경쟁사 5-Layer 역기획

Pass 1 경쟁사 리스트에서 Pain Point 자주 언급된 AI오투오, HubSpot SMB, Mailchimp 선별 (Pain #1,3,10,17 등 AI오투오; Pain #7,18,25 HubSpot; Pain #4,16,23 Mailchimp).

경쟁사 1: AI오투오 (http://o2osolution.ai)

Layer 1 — IA (Information Architecture)

AI오투오
├── 홈
│   ├── 솔루션 소개
│   ├── 데모 신청
│   └── 뉴스/보도자료
├── 서비스
│   ├── AI 분석
│   ├── 콘텐츠 생성
│   ├── SEO/AEO 배포
│   └── 성과 추적
├── 가격
│   ├── 플랜 비교
│   └── 무료 체험
└── 회사
    ├── 팀 소개
    └── 문의
  • 메인 네비게이션은 홈, 서비스, 가격, 회사 4개로 간소화되어 있으며, 서비스 하위에 분석/생성/배포/추적 4개 기능이 명확히 나뉨. 한국 소상공인 타겟에 맞춰 업종별 예시(숙박/병원)가 홈 상단에 배치.
  • 전체 메뉴 수는 12개 정도로 추정되며, 핵심 기능(콘텐츠 생성)까지 클릭 뎁스는 2단계(서비스 > 콘텐츠 생성)로 얕음. 데모 신청은 1클릭으로 접근 가능.
  • 사용자가 가장 자주 쓸 경로: (1) 홈 > 데모 신청 (초기 체험), (2) 로그인 후 서비스 > AI 분석 > URL 입력 (Pain #3 해결 시도), (3) 콘텐츠 생성 > 업종 선택 > 배포 (Pain #1,17 순차 흐름).
  • 소상공인 관점에서 IA 복잡도는 낮음(메뉴 4개, 아이콘 중심). 이해도는 높으나, 성과 추적 뎁스가 깊어(서비스 > 추적 > 대시보드 설정) 1인 사장님 초보자가 헤맬 수 있음. 업종 맞춤 메뉴가 부족해 뷰티/카페 페르소나가 즉시 공감 어려움[1].
  • 추가로 푸터에 정책/문의가 있어 신뢰성 더함, 하지만 모바일에서 서비스 하위 메뉴 접힘으로 뎁스 인지 어려움.

Layer 2 — User Flow (최소 15줄)
- 가입: 이메일/카카오 로그인 1단계, 즉시 무료 체험 URL 입력 유도(허들: 연락처 동의 팝업).
- 온보딩: URL 입력 > 업종 선택 > AI 분석 리포트 3단계(허들: 업종 목록 스크롤 길음, Pain #17 관련).
- 핵심 기능: 분석 리포트 확인 > 콘텐츠 생성 클릭 > 템플릿 선택 4단계(허들: 생성 전 워터마크 표시로 유료 유도).
- 결제: 생성 5회 제한 후 가격 페이지 팝업 1클릭(허들: 카드 입력 필수, 소상공인 예산 Pain #7 트리거).
- 추정 아하 모먼트: 첫 멀티모달 콘텐츠(영상+음악) 생성 완료 후 미리보기 확인 시("이게 자동으로?"), Pain #11 해결 느낌.
- 아하 모먼트 도달 시간: 5-10분 (가입+온보딩+첫 생성).
- 주요 이탈 가능 지점 1: 온보딩 업종 선택(원인: 뷰티/카페 옵션 부족, Pain #17).
- 주요 이탈 가능 지점 2: 생성 후 배포 단계(원인: SEO 설정 복잡, Pain #10).
- 주요 이탈 가능 지점 3: 성과 추적 대시보드(원인: 데이터 연동 수동, Pain #5).
- 가격 티어 사이 전환 허들: Free(5회 생성) → Starter(월 9.9만 원, 무제한 생성)에서 배포 채널 제한(인스타만)이 점프 지점, 추가 채널 유료.
- Free 체험 후 리포트 저장 시 유료 팝업 빈번, 소상공인 Inertia 높임.
- 전체 흐름 7단계로 짧으나, 배포 후 추적 루프 미완성으로 반복 사용 저조 추정.
- 카페 사장 페르소나 기준, 네이버 플레이스 연동 없어 Pull 약함[1].
