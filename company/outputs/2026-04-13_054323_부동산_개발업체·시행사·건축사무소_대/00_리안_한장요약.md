건축사·시행사·개발업체가 토지 하나 볼 때마다 변호사한테 500~1,000만 원 내던 걸, 월 300만 원 구독으로 5분 만에 자동 판단해주는 서비스.

## 🧭 2. 사업 성격 분류
**유형**: 하이브리드

초기 3년은 월 구독료로 꾸준히 현금 뽑고, 3~5년 후 직방·빅밸류·네이버 같은 대형 부동산 플랫폼에 매각하는 경로.
분류 근거: B2B 월 구독이라 현금흐름 안정적이고, 데이터가 쌓일수록 매각 가치가 올라가는 구조.

## 🎯 3. 누구한테 팔 건가
- **페르소나 1**: 건축사무소 실장, 40대, 클라이언트 10곳 담당. 토지 하나 볼 때마다 법무사 부르느라 건당 500~1,000만 원 씀. 한 달에 2~3건.
- **페르소나 2**: 시행사 사업기획팀장, 35~45세, 연간 토지 20건 검토. 특례법 틀리면 10억 날리는 게 진짜 공포. 컨설팅비로 연 1억 이상 지출 중.

## 💢 4. 진짜 고통 (3~5개)
- **특례법 판단**: 이 토지에 특례법 적용되는지 모르면 사업 자체가 날아감 → 지금 법무사·변호사 불러서 해결 → 건당 500~1,000만 원, 3~5일 소요
- **인허가 승인 될지 모름**: 허가 안 날 땅 샀다가 10억 날리는 사례 실재 → 지금 컨설팅 업체에 맡김 → 건당 2,000만 원
- **용적률·건폐율 계산**: 땅마다 지자체 포털 뒤져서 엑셀로 수작업 → 팀원 1명이 주 20시간 씀 → 월 인건비 400만 원 낭비
- **개발 가능성 보고서 작성**: 클라이언트한테 제안서 내야 하는데 외주 맡기면 1주일, 800만 원 → 그 사이 다른 건축사한테 일 뺏김
- **정책 바뀔 때마다 재계산**: 특례법 개정되면 기존 분석 전부 다시 해야 함 → 뉴스 모니터링 팀원 따로 두거나 주 5시간 직접 확인

## 🏆 6. 왜 이길 수 있나
1. **빅밸류는 시세만 봄** ← 우리는 특례법 해당 여부까지 자동 판단. 빅밸류 쓰는 고객도 결국 법무사 따로 부름
2. **직방 프로는 입지 분석만 됨** ← 우리는 토지 개요서 올리면 법적 개발 가능성까지 5분 내 결과. 직방으로 입지 보고 우리로 법적 검토하는 조합 가능
3. **국토연구원 자료는 보고서 형식** ← 직접 토지 정보 입력해서 즉시 결과 받는 자동화 툴이 시장에 없음. 지금이 선점 타이밍

## 🔧 7. 회사 자산 활용도
**활용도 점수**: 5/10

- **B2B 영업 네트워크**: 리안이 기존에 거래하던 건축사·시행사 있으면 첫 5명 영업 바로 가능
- **온라인팀**: 랜딩 페이지·데모 영상 제작에 투입 가능
- **CRM**: 초기 고객 관리·팔로업에 기존 CRM 그대로 활용
- **마케팅팀**: 네이버 카페(부동산 스터디, 건축사 카페, 시행사 카페) 콘텐츠 배포

활용도 5점인 이유: 핵심 자산인 특례법 DB 구축과 AI 개발은 외주 의존. 리안 기존 자산은 영업·마케팅에만 쓸 수 있고 제품 자체는 새로 만들어야 함.

## 🚪 11. 첫 고객 확보 경로 (구체적)

- **다음 50명**: 네이버 카페 "부동산 스터디"·"건축사 카페"에 실제 분석 사례 후기 게시 + 건축사협회 온라인 커뮤니티 직접 접근. 기존 고객 1명당 지인 2명 소개 인센티브(1개월 무료)
- **다음 500명**: 건축사협회·한국시행사협회 공식 파트너십 + 유료 고객 사례 인터뷰 콘텐츠로 네이버 블로그 SEO. 이 시점엔 입소문이 주 채널

## ⚠️ 12. 가장 큰 리스크 3가지 + 망하면 잃는 것
- **리스크 1**: 특례법 자동 판단이 틀렸을 때 고객이 손해 봄 → 대응: 서비스 약관에 "참고용 정보, 법적 효력 없음" 명시 + 변호사 약관 검토 후 출시. 베타 기간엔 전문가 수동 검수 병행
- **리스크 2**: 직방·빅밸류가 6개월 내 유사 기능 출시 → 대응: 지금 당장 고객 데이터·특례법 DB 선점. 고객 연간 계약으로 묶어두기
- **리스크 3**: 건축사·시행사가 "좋다"고 하면서 실제 결제는 안 하는 상황 → 대응: 개발 시작 전에 선결제 3명 확보를 조건으로 걸기. 말만 하는 사람은 고객 아님

**망하면 잃는 것**: 약 8,000만 원 + 6~12개월 + 리안 네트워크 일부 소진

## 🔥 13. 가장 어려운 1가지 (Hard Part)
특례법 자동 판단 결과가 실제로 맞아야 한다 — 한 번 틀리면 시장 전체 신뢰가 날아감.

**첫 행동**: 출시 전 건축사 2명을 "검수 파트너"로 섭외해서 AI 결과를 사람이 교차 확인하는 구조 먼저 만들기.

- **6개월 시점**: 월 매출 3,000만 원 미만이면 접어라 (고객 15사 기준)
- **사전 약속**: 이 숫자 안 나오면 감정 빼고 접기. 이미 쓴 돈은 잊기. 그 시간에 다른 거 하는 게 이득

## 🏁 15. 출구 전략 (Exit)
- **언제**: 약 3~5년 후
- **어떻게**: M&A (전략적 인수)
- **누구한테 팔 가능성**:
  1. **직방** — 부동산 플랫폼 확장 전략상 개발 사업성 분석 툴 필요. 다방 인수 전례 있음
  2. **빅밸류** — 시세 분석에 법적 판단 기능 붙이면 완성형 프롭테크. 우리 기술이 딱 맞는 퍼즐 조각
  3. **네이버 부동산** — 부동산 서비스 고도화 투자 지속 중. B2B 전문 툴 인수로 차별화 가능
- **예상 금액**: 약 300억 ~ 1,000억 원 (연 매출 100억 기준 10배 멀티플)

## 🚀 16. 내일부터 뭐 할까
1. 리안 아는 건축사·시행사 지인 10명 리스트 뽑아서 내일 당장 카카오톡 or 전화. 스크립트: "특례법 해당 여부 5분 만에 자동 판단해주는 거, 월 100만에 베타로 먼저 써볼래요?"
2. 노션 페이지 하나 만들어서 서비스 개요 + 구글폼(관심 등록) 링크 달기 — 코딩 없이 오늘 2시간이면 됨
3. 변호사 1명 섭외해서 "자동 판단 결과 틀렸을 때 법적 책임" 약관 검토 의뢰 — 이거 없으면 유료 판매 못 함
4. 개발 외주사 2~3곳 견적 받기 — MVP 범위는 딱 하나, 특례법 자동 체크만. 나머지는 나중에
5. 3개월 시점 피벗 기준(유료 3사)을 팀 전체와 공유하고 문서로 남기기
