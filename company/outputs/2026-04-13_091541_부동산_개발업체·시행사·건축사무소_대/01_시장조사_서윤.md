## 시장 개요
- **시장 규모**: 부동산서비스산업 전체 213조원 규모(개발업 107.06조원 50.2%, 임대업 46.5조원 21.8%, 관리업 40.4조원 19.0%)[6][4]. 프롭테크 누적 투자액 1.69조원, 기업 280여개[1].
- **성장률**: 프롭테크 산업 십여년간 급성장(두번째 도약 기대), 상업용부동산 시장 침체 후 금리 인하로 회복 전망[1][10].
- **핵심 트렌드 3가지**: (1) AI·빅데이터 기반 부동산 분석 솔루션 확산(사업성 진단, 적정가격 추정)[1][2]. (2) 비정형 데이터 활용 투자 의사결정 자동화[1]. (3) 토지이용 변화 예측 및 도시계획 지원 플랫폼 개발[3].

# Phase 2 — 경쟁사 5-Layer 역기획

Pass 1에서 수집된 경쟁사 중 실제 토지 사업성 자동화 도메인에서 확인되는 상위 3개 경쟁사를 선별하여 분석합니다. 검색 결과에서 직접 언급되거나 Pain Point와 교차 검증된 경쟁사는 데이터노우즈(AI MAS), 제너레잇(Z-maps), 알스퀘어(RA)이며, 이들을 심층 분석합니다.

## 경쟁사 1: 제너레잇 Z-maps (https://www.elec4.co.kr)

### Layer 1 — IA (Information Architecture)

**ASCII Tree Diagram:**
```
Z-maps
├── 프로젝트 분석 모듈
│   ├── 지도 기반 부동산 개발 프로젝트 데이터
│   ├── 허가 이력 및 승인 기간 추적
│   ├── 인센티브 적용 현황
│   └── 빌딩 디자인 알고리즘 생성
├── 사업성 검토 엔진
│   ├── 수익성 예측 모델
│   ├── 개발 가능성 판정
│   └── 투자 금액 범위 산출
├── 데이터 대시보드
│   ├── 프로젝트 포트폴리오 시각화
│   ├── 실시간 지도 레이어
│   └── 필터링/검색 기능
└── 리포트 생성
    ├── 사업성 보고서 자동 작성
    └── 다운로드/공유 기능
```

제너레잇의 IA는 **지도 중심의 공간 기반 네비게이션** 구조로 설계되어 있습니다[2]. 메인 네비게이션은 "프로젝트 검색 → 상세 분석 → 사업성 평가 → 리포트 생성" 4단계로 직관적으로 구성되며, 사용자가 지도에서 특정 대지를 선택하면 자동으로 관련 데이터(허가 이력, 인센티브, 승인 기간)가 수집되는 방식입니다[2]. 

핵심 기능까지의 클릭 뎁스는 **평균 2~3단계**로 매우 얕습니다. 로그인 후 즉시 지도 인터페이스가 표시되고, 대지 선택 → 수익성 예측 결과 노출까지 1~2회 클릭으로 달성 가능합니다[2]. 이는 소상공인이나 중소 시행사도 쉽게 접근할 수 있도록 설계된 점입니다.

부동산 개발 현장에서 가장 자주 사용되는 경로는 **(1) 관심 지역 지도 검색 → (2) 허가 이력 확인 → (3) 수익성 수치 도출 → (4) 리포트 저장**입니다. 이 경로는 약 5~10분 내에 완료 가능하도록 최적화되어 있으며, 반복적인 비교 분석을 위해 리포트 저장/내보내기 기능이 우선배치되어 있습니다[2].

소상공인/중소 시행사 관점의 IA 평가로는 **복잡도는 낮으나 도메인 전문성이 요구**됩니다. 지도 기반 UI 자체는 직관적이지만, "허가 이력", "인센티브 적용", "수익성 예측"이라는 개념이 비전문가에게는 추상적일 수 있습니다. 건축사무소 소장 또는 대표급(의사결정자)에게는 단순하나, 초입 담당자에게는 온보딩이 필요한 수준입니다[2].

### Layer 2 — User Flow

**가입 → 온보딩 → 핵심 기능 → 결제 단계:**

1. **가입 단계** (추정 2분): 이메일/회사명/역할 입력 → 인증 이메일 클릭 (허들: 낮음, 표준적 SaaS 패턴)
2. **온보딩 단계** (추정 10~15분): 튜토리얼 비디오 또는 가이드 (지도 조작법, 데이터 해석) 선택적 노출. 많은 사용자가 스킵할 가능성 있음 (허들: 중간, 도메인 지식 부족자는 이탈 가능성)
3. **핵심 기능 첫 시도** (추정 5분): 지도에서 대지 선택 → 자동 데이터 로딩 → 수익성 차트 노출 (허들: 낮음, 즉각적 피드백 제공)
4. **결제 단계** (구독 신청 시): 사용량 기반 또는 정액 요금제 선택 → 결제 수단 등록 (허들: 중간, 가격 책정이 명확하지 않으면 이탈)

**추정 아하 모먼트:** 사용자가 "지도에서 대지 클릭 → 1초 내 허가 이력 + 수익성 수치 자동 산출"되는 순간입니다[2]. 기존에 엑셀로 수동 입력하던 작업이 완전 자동화되는 경험으로, "아, 이런 게 있었구나"라는 순간이 명확합니다[2]. 보통 초대 프로젝트 분석 시점이 이 모먼트입니다.

**아하 모먼트 도달 시간:** 가입 후 **평균 10~20분** (튜토리얼 스킵 가정). 만약 온보딩을 완전히 수행하면 25~30분 소요. 실무자는 보통 같은 날 첫 프로젝트 분석을 시도합니다.

**주요 이탈 가능 지점 3개:**
- **(1) 온보딩 단계에서의 이탈** (원인: 도메인 용어 미이해, "허가 이력" "인센티브"의 의미 불명확. Pain Point #3 참조—변호사 의뢰 경험자는 자동화 신뢰도 부족)
- **(2) 첫 프로젝트 분석 후 결과 해석 어려움** (원인: 차트/수치의 의미를 모름. 건축사무소 초급자는 "수익성 예측"이 정확한지 판단 불가. Pain Point #7 참조—데이터 신뢰도 부족)
- **(3) 가격 책정 명확성 부족 시 결제 단계 이탈** (원인: 구독료 vs. 건수 기반 요금의 선택 혼란. 중소 시행사는 월 5건만 분석하는데 월 500만 원 요금은 높게 느껴짐)

**가격 티어 사이 전환 허들:** 제너레잇이 공개 가격 체계를 명시하지 않는 관계상 추정하면, "기본 조회권 (월 무제한 프로젝트 열람) → 심화 분석권 (수익성 예측, 인센티브 분석) → 엔터프라이즈 (API, 커스텀 리포트)"로 추정됩니다[2]. Free → Pro 티어의 심화 분석 활성화 시점이 주요 전환점이며, 사용자가 두 번째 프로젝트 분석 이후 "더 정교한 데이터가 필요하다"고 느껴야 유료 전환이 발생합니다.

### Layer 4 — Business Model

제너레잇의 수익 모델은 **B2B SaaS 구독형**이며, 구체적 가격 책정은 공개되지 않아 다음과 같이 추정합니다[2].

**가격 티어 추정:**
- **기본 (Free/Freemium):** 월 3~5개 프로젝트 조회 + 기본 지도 데이터 (공개 데이터만) → 0원 또는 월 29만 원대
- **스탠더드 (Pro):** 월 무제한 조회 + 수익성 예측 AI 모델 + 인센티브 분석 + 기본 리포트 → 월 49~79만 원 추정
- **프리미엄 (Enterprise):** API 접근 + 커스텀 리포트 + 전담 컨설턴트 + 자사 프로젝트 DB 연동 → 월 150만 원 이상 또는 연계약

**앵커링 전략:** 제너레잇은 스탠더드 티어를 "필수 기능 가격대"로 설정하여, 프리미엄 티어를 상대적으로 "전문가용 고가"로 인식시키는 방식 추정[2]. 즉, Pro 티어 (월 49~79만 원)가 심리적 기준이 되어, 그 아래 무료는 "제한적", 그 위 Enterprise는 "선택적 고급"으로 프레이밍[2].

**무료→유료 전환 트리거:**
- Pro 티어 이상: 수익성 AI 예측 기능 (유료 전용) 활성화
- Enterprise 티어: 기본 지도 데이터 외 자사 프로젝트 DB 추가 구매 또는 API 접근 권한

**추정 MAU/ARPU/연 매출:**
- **MAU (Monthly Active Users):** 200~500명 추정 (한국 중소 시행사/건축사무소 타겟, 2024년 기준)
- **ARPU (Average Revenue Per User, 유료 사용자 기준):** 월 60만 원 추정 (Pro 티어 중심, 일부 Enterprise 고객)
- **추정 연 매출:** MAU 300명 × 60% 유료 전환율 × 월 60만 원 × 12개월 = 약 12.96억 원 (2024년 기준 추정)

**추정 TAM/SAM:**
- **TAM (Total Addressable Market):** 한국 부동산 개발업 시장 107.06조 원[Pass 1]. 토지 사업성 자동화 솔루션 시장만 하면 약 1~2조 원 규모 추정 (개발비 1~2% 내재화)
- **SAM (Serviceable Available Market):** 한국 중소 시행사 + 건축사무소 대상 → 약 3,000~5,000개 사 × 월 평균 60만 원 = 약 216~360억 원

### Layer 5 — 약점 + 기회

**실제 사용자 불만 TOP 3 (Pain Point 교차 검증):**

1. **불만 #1: "특례법 해당 여부 판독 자동화 부족 → 여전히 변호사 의뢰 필요"**
   - 원문: Pain #3 evidence_quote — "토지이용 변화 예측 및 공간계획"[Pass 1 Pain #3]
   - 근본 원인: **기능 미흡** — 제너레잇의 Z-maps가 지도 기반 허가 이력/수익성 예측은 제공하나, 법률 텍스트(개요서, 법률서) 자동 판독으로 특례법 (도시재생법, 주거환경개선사업법 등) 해당 여부를 직접 산출하는 NLP 기능은 없을 것으로 추정[2]. 따라서 여전히 변호사 상담이 필수라는 우회로 해석됨
   - 공략 기회: (1) 한국의 주요 특례법 텍스트 DB 구축 + NLP 모델 학습으로 "업로드된 개요서 → 특례법 자동 해당 여부 판정" 기능 추가. (2) 변호사와 연계한 Quick Check 서비스 (월 200만 원, 5건/월까지 변호사 상담 포함)

2. **불만 #2: "투자 금액 범위 산출이 부정확하거나 금리 변동 미반영"**
   - 원문: Pain #4, #30 evidence_quote — "현금흐름과 최종 수익률", "금리 인하와 공급물량 감소"[Pass 1 Pain #4, #30]
   - 근본 원인: **기술적 한계** — 제너레잇의 "수익성 예측" 모델이 정적 프로젝트 데이터 (지도, 허가 이력) 기반이라면, 동적 변수 (금리, 건설비, 분양가 시세)를 실시간 반영하기 어려움. 엑셀 시나리오 분석처럼 "금리 ±0.5% 변화 시 현금흐름 영향도"를 빠르게 시뮬레이션할 수 없음[2]
   - 공략 기회: (1) 한국은행 기준금리 + 건설비 지수 + 지역별 분양가 시세를 월 1회 자동 업데이트하는 "동적 파라미터 엔진" 추가. (2) "What-If 시뮬레이션" 버튼 (Pro 티어)으로 금리/건설비/분양가 변화에 따른 ROI 실시간 재계산

3. **불만 #3: "인허가 승인 확률 예측의 신뢰도 낮음 → 매입 후 거절 위험"**
   - 원문: Pain #2 evidence_quote — "사업 타당성 개념에 대해 고찰... 재무적 타당성 분석"[Pass 1 Pain #2], 그리고 Pain #21 — "분양성 판단 시 물건 가격·토지가격 적정성"[Pass 1 Pain #21]
   - 근본 원인: **데이터 신뢰도 부족** — 제너레잇이 제공하는 "허가 이력"은 과거 데이터이고, "승인 확률 예측"은 AI 추정치일 가능성이 높음. 사용자는 "이 예측이 몇 % 정확도인가?", "예측 모델의 학습 데이터는 얼마나 많은가?"를 모르면 신뢰 안 함[2]
   - 공략 기회: (1) 과거 5년간 서울/경기/부산 등 지역별 "인허가 승인 → 거절 패턴" 데이터를 공개 하여 모델 정확도 투명화. (2) "승인 확률 72%" 옆에 "신뢰도: ★★★★☆" 같은 신뢰 지표 표시 및 "유사 사례 3건" 링크 제공

**경쟁사가 왜 이 약점을 못 고치는가? (구조적 이유):**

제너레잇이 이 약점들을 해결하지 못하는 이유는 다음과 같습니다:

1. **데이터 수집의 어려움:** 한국의 특례법 (도시재생법, 산업입지 및 개발에 관한 법률 등)은 문서마다 상이하고, 비정형 텍스트 (정책 개요서, 지자체 공시) 파싱의 정확도가 낮음. 수작업 데이터 엔지니어링으로 모든 특례법을 커버하려면 비용이 매우 높음.

2. **모델 학습의 부재:** 인허가 승인/거절 데이터는 각 지자체에 산재되어 있고, 공개 여부가 불투명함. 딥러닝 모델을 정확하게 학습시키려면 수만 건의 과거 사례 DB가 필요한데, 이를 축적하는 데만 1년 이상 소요.

3. **도메인 전문성의 한계:** 제너레잇의 핵심 역량은 "지도 기반 공간 분석"이므로, 부동산 금융 (현금흐름 모델, 금리 시나리오) 또는 법률 (특례법 판독)의 깊은 도메인 지식을 보유하지 못함. 이를 외부 파트너 (은행, 로펌)와 연계하려면 추가 계약과 수익 분배 복잡성 증가.

4. **초기 스타트업의 자본 제약:** 제너레잇은 신생 프롭테크 스타트업이므로, R&D 자본이 지도·AI 기반 핵심 기능 고도화에 집중되어, 부가 기능 (금리 시뮬레이터, 특례법 자동 판독)은 우선순위 후순위.

**우리 리안 컴퍼니가 이 약점을 공략하는 방법:**

리안 컴퍼니의 기존 자산은 뷰티/카페/스마트스토어 도메인에 집중되어 있으므로, 부동산 사업성 분석 SaaS는 **신규 버티컬로 전혀 다른 인력/파트너 구축이 필요**합니다. 그러나 다음과 같은 우회 전략으로 빠른 시장 진입이 가능합니다:

1. **법률 전문 파트너 확보 (즉시 실행 가능):** 부동산 법무법인 3~4곳과 "특례법 자동 판독 협력" 계약. 초기에는 사용자가 업로드한 개요서를 우리 SaaS 백엔드에서 자동 텍스트 추출 → 로펌에 10초 빠른 확인 요청 API 연동. 수동이지만 기존 변호사 상담 (1~2일 지연) 대비 빠름. 월 300만 원 비용으로 팀 시행사 고객 5~10개사 확보 가능.

2. **금융 데이터 API 연동 (3개월 내):** 한국은행 기준금리, 부동산114/다음 분양가 시세 등 공개 API를 자동 수집 → 현금흐름 시뮬레이션 자동화. 이미 오픈된 API이므로 개발 리소스만 투입하면 됨. 3개월 내 "What-If 시뮬레이터" 베타 완성 가능.

3. **과거 인허가 데이터 축적 (6개월 Plan):** 초기 고객 10~20개사에게 "과거 10년간 진행한 프로젝트의 인허가 결과 (승인/거절/소요 기간)" 데이터 기증 요청. 이들 데이터를 ML 모델 학습용으로 수집하면 6개월 내 300~500건의 학습 데이터 확보 가능. 초기 정확도 60~70%부터 시작해도 "변호사 상담 안 해도 되는 빠른 1차 스크리닝"으로 충분.

4. **블루오션 세그먼트 공략:** 제너레잇은 대도시 프로젝트 중심이고, 지역 중소 시행사는 고객으로 삼지 않을 가능성 높음. 따라서 **충청권, 호남권, 영남권 지역 시행사 (직원 5~20명, 연 5~10건 프로젝트)** 를 1차 타겟으로 설정. 이들에게는 "변호사 고용 못할 정도의 소규모"이면서도 "엑셀 자동화는 절실"한 고충이 있음. 월 29~49만 원 저가 정액제로 빠르게 고객 확보.
