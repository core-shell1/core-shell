[9] Flywheel + Engine of Growth

Flywheel 초안 (6개 요소)

1. 사용자가 토지 1건 업로드
     ↓
2. 48시간 내 승인 확률 78% + 유사 사례 3건 리포트 받음
     ↓
3. "정확하네!" 만족 → 2건째 분석 요청 (유료 전환)
     ↓
4. 월 10건 분석 → 과거 데이터 축적 (우리 DB 강화)
     ↓
5. ML 모델 정확도 향상 (75% → 80%) → 더 정확한 예측
     ↓
6. 만족한 고객이 동료 대표에게 추천 ("이거 진짜 좋아")
     ↑
1. 추천받은 신규 사용자가 토지 1건 업로드 (선순환)

각 연결고리 작동 원리 1줄:
- 1→2: 사용자가 개요서 PDF 업로드 → 우리 시스템이 OCR + ML 모델 → 48시간 내 리포트 자동 생성 → 이메일 발송
- 2→3: 리포트에서 "승인 확률 78% + 유사 사례 3건" 확인 → "컨설턴트보다 빠르고 정확하네!" 만족 → 2건째 분석 시도 (유료 전환 트리거)
- 3→4: 유료 고객이 월 평균 10건 분석 → 우리 DB에 10건 × 고객 50명 = 월 500건 데이터 축적 → ML 모델 학습 데이터 증가
- 4→5: 과거 데이터 300건 → 800건 → 1,300건 증가 → ML 모델 재학습 → 정확도 75% → 80% 향상 → 더 정확한 예측 제공
- 5→6: 정확도 80% 달성 → 사용자가 "이거 진짜 정확해!" 만족 → 동료 대표 (골프 모임, 세미나)에게 "너도 써봐" 추천 → 바이럴 계수 1.2
- 6→1: 추천받은 신규 사용자가 14일 무료 체험 가입 → 토지 1건 업로드 → 선순환 시작

Engine of Growth 선택

선택: Sticky (리텐션 기반)

근거:
- B2B SaaS 특성상 사용자가 한 번 익숙해지면 Switching Cost 높음 (엑셀 → SaaS 전환 시 학습 곡선)
- 인허가 승인 확률 예측은 반복적 니즈 (연 10~20건 토지 검토) → 매월 사용
- 우리 가치 제안 "48시간 내 리포트 + 정확도 75% 보장"은 대체재 없음 (컨설턴트는 2주 소요 + 비용 높음) → 사용할수록 의존도 증가

가속 트리거:
- ML 모델 정확도 향상 (75% → 80% → 85%) — 사용자가 많을수록 데이터 축적 → 정확도 향상 → 만족도 증가 → Churn 감소
- 구체적: 고객 50명 × 월 10건 = 월 500건 데이터 축적 → 3개월마다 ML 모델 재학습 → 정확도 +5%p 향상

해자 (Moat) 가능성:
- 데이터 해자 (Data Moat): 과거 인허가 승인/거절 데이터 300건 → 800건 → 1,300건 축적 → 경쟁사는 데이터 없음 → 6개월 격차
- 파트너 해자 (Partner Moat): 법무법인 3곳과 독점 계약 (API 연동) → 경쟁사는 로펌 섭외 어려움 (법률 도메인 진입장벽)
- 네트워크 효과 (약함): 사용자가 많을수록 유사 사례 DB 풍부 → 예측 정확도 향상 → 신규 사용자 유입 증가 (단, B2B 특성상 네트워크 효과는 제한적)

해자 없는 경우 대응:
- 만약 경쟁사 (제너레잇 등)가 데이터 + 파트너 빠르게 확보 시 → 실행 속도로 대체
- 구체적: 우리는 6개월 내 MVP 출시 + 베타 고객 15개사 확보 → 경쟁사는 기획~개발 1년 소요 → 6개월 선점 효과
- 선점 효과로 고객 50개사 확보 → 데이터 500건 축적 → 경쟁사가 따라잡으려면 추가 6개월 필요 → 총 1년 격차 유지

제너레잇의 IA는 지도 중심의 공간 기반 네비게이션 구조로 설계되어 있습니다[2]. 메인 네비게이션은 "프로젝트 검색 → 상세 분석 → 사업성 평가 → 리포트 생성" 4단계로 직관적으로 구성되며, 사용자가 지도에서 특정 대지를 선택하면 자동으로 관련 데이터(허가 이력, 인센티브, 승인 기간)가 수집되는 방식입니다[2].

핵심 기능까지의 클릭 뎁스는 평균 2~3단계로 매우 얕습니다. 로그인 후 즉시 지도 인터페이스가 표시되고, 대지 선택 → 수익성 예측 결과 노출까지 1~2회 클릭으로 달성 가능합니다[2]. 이는 소상공인이나 중소 시행사도 쉽게 접근할 수 있도록 설계된 점입니다.

부동산 개발 현장에서 가장 자주 사용되는 경로는 (1) 관심 지역 지도 검색 → (2) 허가 이력 확인 → (3) 수익성 수치 도출 → (4) 리포트 저장입니다. 이 경로는 약 5~10분 내에 완료 가능하도록 최적화되어 있으며, 반복적인 비교 분석을 위해 리포트 저장/내보내기 기능이 우선배치되어 있습니다[2].

소상공인/중소 시행사 관점의 IA 평가로는 복잡도는 낮으나 도메인 전문성이 요구됩니다. 지도 기반 UI 자체는 직관적이지만, "허가 이력", "인센티브 적용", "수익성 예측"이라는 개념이 비전문가에게는 추상적일 수 있습니다. 건축사무소 소장 또는 대표급(의사결정자)에게는 단순하나, 초입 담당자에게는 온보딩이 필요한 수준입니다[2].

Layer 2 — User Flow

가입 → 온보딩 → 핵심 기능 → 결제 단계:

  1. 가입 단계 (추정 2분): 이메일/회사명/역할 입력 → 인증 이메일 클릭 (허들: 낮음, 표준적 SaaS 패턴)
  2. 온보딩 단계 (추정 10~15분): 튜토리얼 비디오 또는 가이드 (지도 조작법, 데이터 해석) 선택적 노출. 많은 사용자가 스킵할 가능성 있음 (허들: 중간, 도메인 지식 부족자는 이탈 가능성)
  3. 핵심 기능 첫 시도 (추정 5분): 지도에서 대지 선택 → 자동 데이터 로딩 → 수익성 차트 노출 (허들: 낮음, 즉각적 피드백 제공)
  4. 결제 단계 (구독 신청 시): 사용량 기반 또는 정액 요금제 선택 → 결제 수단 등록 (허들: 중간, 가격 책정이 명확하지 않으면 이탈)

추정 아하 모먼트: 사용자가 "지도에서 대지 클릭 → 1초 내 허가 이력 + 수익성 수치 자동 산출"되는 순간입니다[2]. 기존에 엑셀로 수동 입력하던 작업이 완전 자동화되는 경험으로, "아, 이런 게 있었구나"라는 순간이 명확합니다[2]. 보통 초대 프로젝트 분석 시점이 이 모먼트입니다.

아하 모먼트 도달 시간: 가입 후 평균 10~20분 (튜토리얼 스킵 가정). 만약 온보딩을 완전히 수행하면 25~30분 소요. 실무자는 보통 같은 날 첫 프로젝트 분석을 시도합니다.

주요 이탈 가능 지점 3개:
- (1) 온보딩 단계에서의 이탈 (원인: 도메인 용어 미이해, "허가 이력" "인센티브"의 의미 불명확. Pain Point #3 참조—변호사 의뢰 경험자는 자동화 신뢰도 부족)
- (2) 첫 프로젝트 분석 후 결과 해석 어려움 (원인: 차트/수치의 의미를 모름. 건축사무소 초급자는 "수익성 예측"이 정확한지 판단 불가. Pain Point #7 참조—데이터 신뢰도 부족)
- (3) 가격 책정 명확성 부족 시 결제 단계 이탈 (원인: 구독료 vs. 건수 기반 요금의 선택 혼란. 중소 시행사는 월 5건만 분석하는데 월 500만 원 요금은 높게 느껴짐)

가격 티어 사이 전환 허들: 제너레잇이 공개 가격 체계를 명시하지 않는 관계상 추정하면, "기본 조회권 (월 무제한 프로젝트 열람) → 심화 분석권 (수익성 예측, 인센티브 분석) → 엔터프라이즈 (API, 커스텀 리포트)"로 추정됩니다[2]. Free → Pro 티어의 심화 분석 활성화 시점이 주요 전환점이며, 사용자가 두 번째 프로젝트 분석 이후 "더 정교한 데이터가 필요하다"고 느껴야 유료 전환이 발생합니다.

경쟁사가 왜 이 약점을 못 고치는가? (구조적 이유):

제너레잇이 이 약점들을 해결하지 못하는 이유는 다음과 같습니다:

  1. 데이터 수집의 어려움: 한국의 특례법 (도시재생법, 산업입지 및 개발에 관한 법률 등)은 문서마다 상이하고, 비정형 텍스트 (정책 개요서, 지자체 공시) 파싱의 정확도가 낮음. 수작업 데이터 엔지니어링으로 모든 특례법을 커버하려면 비용이 매우 높음.

  2. 모델 학습의 부재: 인허가 승인/거절 데이터는 각 지자체에 산재되어 있고, 공개 여부가 불투명함. 딥러닝 모델을 정확하게 학습시키려면 수만 건의 과거 사례 DB가 필요한데, 이를 축적하는 데만 1년 이상 소요.

  3. 도메인 전문성의 한계: 제너레잇의 핵심 역량은 "지도 기반 공간 분석"이므로, 부동산 금융 (현금흐름 모델, 금리 시나리오) 또는 법률 (특례법 판독)의 깊은 도메인 지식을 보유하지 못함. 이를 외부 파트너 (은행, 로펌)와 연계하려면 추가 계약과 수익 분배 복잡성 증가.

  4. 초기 스타트업의 자본 제약: 제너레잇은 신생 프롭테크 스타트업이므로, R&D 자본이 지도·AI 기반 핵심 기능 고도화에 집중되어, 부가 기능 (금리 시뮬레이터, 특례법 자동 판독)은 우선순위 후순위.

우리 리안 컴퍼니가 이 약점을 공략하는 방법:

리안 컴퍼니의 기존 자산은 뷰티/카페/스마트스토어 도메인에 집중되어 있으므로, 부동산 사업성 분석 SaaS는 신규 버티컬로 전혀 다른 인력/파트너 구축이 필요합니다. 그러나 다음과 같은 우회 전략으로 빠른 시장 진입이 가능합니다:

  1. 법률 전문 파트너 확보 (즉시 실행 가능): 부동산 법무법인 3~4곳과 "특례법 자동 판독 협력" 계약. 초기에는 사용자가 업로드한 개요서를 우리 SaaS 백엔드에서 자동 텍스트 추출 → 로펌에 10초 빠른 확인 요청 API 연동. 수동이지만 기존 변호사 상담 (1~2일 지연) 대비 빠름. 월 300만 원 비용으로 팀 시행사 고객 5~10개사 확보 가능.

  2. 금융 데이터 API 연동 (3개월 내): 한국은행 기준금리, 부동산114/다음 분양가 시세 등 공개 API를 자동 수집 → 현금흐름 시뮬레이션 자동화. 이미 오픈된 API이므로 개발 리소스만 투입하면 됨. 3개월 내 "What-If 시뮬레이터" 베타 완성 가능.

  3. 과거 인허가 데이터 축적 (6개월 Plan): 초기 고객 10~20개사에게 "과거 10년간 진행한 프로젝트의 인허가 결과 (승인/거절/소요 기간)" 데이터 기증 요청. 이들 데이터를 ML 모델 학습용으로 수집하면 6개월 내 300~500건의 학습 데이터 확보 가능. 초기 정확도 60~70%부터 시작해도 "변호사 상담 안 해도 되는 빠른 1차 스크리닝"으로 충분.

  4. 블루오션 세그먼트 공략: 제너레잇은 대도시 프로젝트 중심이고, 지역 중소 시행사는 고객으로 삼지 않을 가능성 높음. 따라서 충청권, 호남권, 영남권 지역 시행사 (직원 5~20명, 연 5~10건 프로젝트) 를 1차 타겟으로 설정. 이들에게는 "변호사 고용 못할 정도의 소규모"이면서도 "엑셀 자동화는 절실"한 고충이 있음. 월 29~49만 원 저가 정액제로 빠르게 고객 확보.
