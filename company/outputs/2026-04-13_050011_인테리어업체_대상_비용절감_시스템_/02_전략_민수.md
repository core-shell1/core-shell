## 1. Pain Point 필터링 (돈이 되는 고통만 남기기)
서윤의 Pain Point 중:
- 1차: Level 4~5만 통과 (Level 1~3 제거)
- 2차: 임시방편 사용 중(Workaround) 최우선
- 3차: 출처 신뢰도 낮은 것 폐기
- 블루오션 플래그: Level 4~5인데 경쟁자 없는 것 별도 표기

→ 통과/탈락 수 + 블루오션 플래그 목록 먼저 보여줘
- 통과: 15개
- 탈락: 15개
- 블루오션 플래그 목록: 거래망 추천 없음, 법률 리스크 체크 없음, 공기 산출 없음, 사업성 평가서 없음

## 2. TOP 3~5 아이템 선정 + 비교표

| 기준 | 가중치 |
|------|--------|
| Workaround 비용 (이미 돈/시간 쓰는가?) | 20% |
| 지불 주체(Payer) 명확성 | 15% |
| Pre-sell 가능성 (제품 없이 선결제 가능?) | 15% |
| 경쟁 빈틈 (역기획 Gap 일치?) | 25% |
| Build 난이도 (MVP 1~2 기능으로 가치 전달?) | 15% |
| 우리 자산 활용도 | 10% |

비교표:

|------|---------|---------|---------|
| 종합 점수 | 82 | 78 | 76 |
| Pain Level | 5 | 5 | 4 |
| 페르소나 | 구매 담당 | 인테리어 대표 | 시공팀장 |
| Workaround 비용 | 연체 손실 500만 원/년 | 건당 200만 원 | 오류 수정 10시간/건 |
| MVP 핵심 기능 | 신용도 기반 거래망 추천 | 자동 법률 리스크 체크 | 자동 공기 산출 |
| 예상 가격 | 월 20만 원 | 월 25만 원 | 월 15만 원 |

## 3. 1순위 선택 + 포지셔닝
- 1순위 선택: 거래망 추천
- 킬러 오퍼 공식: [구매 담당자] + [신뢰할 수 있는 거래망 추천] + [신용도 기반 매칭 시스템] + [월 20만 원]
- 좋은 예: "신용도 기반으로 안전한 거래망을 추천받고 거래 리스크를 줄이세요 (월 20만 원)"

## 4. 수익 모델 + 가격 전략

## 5. 48시간 검증 실험
- 코딩 없이 수요 검증하는 방법: 랜딩 페이지 + 가짜 문 열기(Fake Door) 테스트
- 성공 판단 기준 (KPI): 50명 이상 연락처 등록, 10명 이상 사전 결제 의사
- 실패 시: 2순위로 피벗 (법률 리스크 체크)

Problem Interview 질문지 (5개):
1. 현재 거래망 관리는 어떻게 하고 계신가요? (과거)
2. 기존 거래망 관리에서 가장 큰 문제가 무엇인가요? (행동)
3. 거래 추천 서비스를 사용해본 적이 있나요? (행동)
4. 이 서비스를 사용한다면 어떤 점이 가장 도움이 될 것 같나요? (행동)
5. 서비스가 출시된다면 바로 사용해보고 싶으신가요? (약속)

## 6. 초기 진입 전략
[구체적 액션 3가지 — "내일 당장 할 수 있는 것" 위주]
1. 거래망 추천 MVP 설계: 핵심 기능 정리 및 우선순위 설정
2. 랜딩 페이지 제작 및 Fake Door 테스트 시작
3. 초기 고객 확보를 위한 직접 아웃리치 시작

## 7. 단계별 실행 계획 (초기/중기/장기)

### 7-1. 초기 (0~3개월) — 시장 진입

| 항목 | 내용 |
|------|------|
| 목표 (측정 가능) | 첫 100명의 유료 고객 확보 |
| 주요 마일스톤 (월 단위) | MVP 출시, 초기 고객 확보, 피드백 수렴 및 개선 |
| 필요 인력 / 예산 | 개발자 2명, 마케팅 1명 / 월 500만 원 |
| 예상 고객수 | 100명 |
| 예상 매출 | 2,000만 원 |
| 성공 지표 + 임계값 | 월 매출 1,000만 원 달성 |
| 병목 / 리스크 | 초기 고객 유입 부족 |
| 대응 시나리오 | 가격 인하 프로모션, 추가 마케팅 캠페인 |

### 7-2. 중기 (3~12개월) — 성장 · 스케일

| 항목 | 내용 |
|------|------|
| 목표 | 월 매출 1억 원 |
| 제품 로드맵 (주요 기능 3~5개) | 법률 리스크 체크, 공기 산출, 사업성 평가서 추가 |
| 확장 전략 (채널/지역/세그먼트) | 대형 시공업체로 확장, 지역별 커뮤니티 타겟 |
| 경쟁 대응 | 가격 경쟁력 확보, 기능 차별화 |
| 조직 확장 (채용 계획) | 추가 개발자 3명, 마케팅 2명 |
| 예상 고객수 / 월매출 | 1,000명 / 1억 원 |
| 운영 효율화 포인트 | 고객 지원 자동화, 내부 프로세스 최적화 |

### 7-3. 장기 (1~3년) — 시장 지배 · Exit 준비

| 항목 | 내용 |
|------|------|
| 목표 | 시장 점유율 30% |
| 경쟁 우위 고착화 전략 (moat) | 경쟁사 대비 차별화된 기술 및 데이터 확보 |
| 시장 점유율 목표 | 30% |
| 연 매출 / 밸류에이션 타겟 | 100억 원 / 1,000억 원 |
| 지적재산권 / 데이터 자산 확보 | 특허 출원 및 데이터베이스 구축 |

## 10. 최적 단가 + 단위 경제학

### 10-1. 가격 티어 (상세)

| 플랜 | 월 가격 | 포함 내용 | 타겟 페르소나 | 업셀 트리거 |
|------|---------|-----------|--------------|-------------|
| Starter | 20만 원 | 신용도 기반 거래망 추천 | 소규모 인테리어 업체 | 법률 리스크 체크 |
| Pro | 35만 원 | Starter + 법률 리스크 체크 | 중형 인테리어 업체 | 공기 산출 |
| Enterprise | 문의 | Pro + 커스텀 세팅 | 대형 인테리어 업체 | 사업성 평가서 |

### 10-2. 단가 근거 (반드시 숫자로)
- 경쟁사 가격 범위: 연 1,200~3,500달러 (약 140만~400만 원)
- 원가 구조: 고정비 40%, 변동비 30%, 한계비용 30%
- 지불 의사 조사 근거: "신용도 기반 거래망 추천 없음" 블루오션 플래그
- Anchor 가격 설계: Pro 티어를 중간 가격으로 설정해 Starter의 가성비 강조
- 업셀/다운셀 경로: Pro로 업셀 유도, 필요 시 Starter로 다운셀

### 10-3. 유닛 이코노믹스 (Unit Economics)

| 지표 | 목표치 | 산정 근거 |
|------|--------|----------|
| CAC (고객 획득 비용) | 5만 원 | 채널별 CPC × 전환율 역산 |
| LTV (생애 가치) | 50만 원 | ARPU × 평균 유지 개월 × 마진율 |
| LTV/CAC 비율 | ≥3 | LTV 50만 원 / CAC 5만 원 |
| Payback Period | ≤18개월 | LTV/CAC 기준 |
| Gross Margin | ≥70% | 고정비 대비 수익 |
| 월 churn율 | ≤5% | 고객 유지율 분석 |

## 핵심 가설 (검증 필요)

| 가설 | 검증 방법 | 기간 |
|------|---------|------|
| 거래망 추천이 수주율을 높인다 | 랜딩 + DM 아웃리치 | 2주 |
| 법률 리스크 체크가 공기 지연을 줄인다 | 설문 + 인터뷰 | 1개월 |
| 자동 공기 산출이 클라이언트 신뢰를 높인다 | 고객 피드백 | 1개월 |

핵심 원칙:
- 이론 금지. 바로 실행 가능한 전략만.
- 우리가 이미 가진 자산을 최대한 활용해라.
- "좋아요 100개"보다 "결제 1건"이 중요하다.
- 만들기 전에 팔아라. 데모로 지불 의사 확인 먼저.
- 초기 10명에게 직접 서비스하고 검증한 후 확장해라.
- 피벗 신호: 핵심 지표 정체, 유료 전환 0, 고객이 다른 문제를 말함.
