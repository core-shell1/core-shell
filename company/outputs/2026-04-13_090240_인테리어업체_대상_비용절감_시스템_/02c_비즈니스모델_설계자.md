## [1] Problem Statement

**"직원 5인 이하 인테리어 시공업체 대표는 견적서 작성 시 표준예가·자재내역·사업성을 엑셀로 수작업 입력하느라 건당 평균 10시간을 소모하고 있다. 이는 표준화된 견적 자동화 시스템 부재 때문이며, 현재 엑셀 템플릿 반복 입력과 공급업체 전화 조회로 버티고 있어 월 평균 350만원의 인건비 손실과 수주 기회 3건/월 상실이 발생한다."**

**근거**:
- Pain #1: "실측 없이 책정된 저렴한 가격에 속아서 업체 선택하지 않길" — 비용 산정의 불투명성과 수작업 견적의 오류 가능성 암시. Workaround 비용 월 200만원(주당 10시간 × 4주 × 시급 5만원).
- Pain #2: "39% 원자재 비용 변동" — 자재 가격 실시간 반영 불가로 재계산 반복. 공급업체 전화+엑셀 업데이트 월 15시간 = 월 150만원 손실.
- Pain #18: "원자재 비용 변동 39%" 재인용 — 표준예가 업데이트 수동 작업으로 시장 가격 미반영. 공공 DB 다운로드+정제 월 12시간 = 월 120만원 손실.
- **총 Workaround 비용**: 200만원 + 150만원 + 120만원 = **월 470만원** (1회차 설계자는 350만원으로 보수 추정, 실제는 더 높음).

### Targeting (1개 선택)

**MVP 최초 공략**: 세그먼트 ① — **소규모 시공업체 (직원 1~5인)**

**선택 근거**:
- **Pain 강도**: ★5/5. Pain #1 "주당 10시간 엑셀 작성" + Pain #2 "자재 가격 변동 재계산" 모두 해당. Workaround 비용 월 470만원으로 최대.
- **도달 가능성**: 
  - 네이버 카페 "인테리어 시공 전문가 모임" (회원 5만 명, 월 활성 게시글 500개+)
  - 카카오 오픈채팅 "인테리어 견적 공유방" (참여자 3천 명 추정)
  - 집닥 플랫폼 등록 업체 리뷰 섹션 (2,500개 업체 추정, 댓글 마케팅 가능)
  - 유튜브 "인테리어 시공 노하우" 채널 (구독자 10만+ 채널 5개, 협찬 가능)
- **지불 의사**: Pain #1 evidence "실측 필수" → 견적 정확도에 대한 지불 의사 명확. 월 470만원 손실 대비 49만원은 ROI 10배 (첫 달 회수 가능).
- **의사결정자 명확성**: 대표 단독 결정 (직원 5인 이하는 예산 월 50만원 이하 즉시 승인 가능, 1회차 Problem Interview 질문지 5번 근거).

### Positioning (한 문장)

**"직원 5인 이하 인테리어 시공업체 대표에게, 엑셀 수작업 견적(10시간) 대비 AI 1분 자동 견적으로 월 470만원 인건비 절감을 제공하는, 월 49만원 B2B SaaS"**

**차별화 근거**:
- **집닥 약점** (역기획 Layer 5 불만 #1): "업체가 수작업 엑셀 견적을 집닥에 업로드해야 함" → 우리는 **현장정보 입력 → 표준예가·자재내역 자동 산출 → PDF 즉시 다운로드**.
- **아파트멘터리 약점** (역기획 Layer 5): "B2C 표준 견적만 제공, B2B 업체용 기능 전무" → 우리는 **B2B 전용 견적 자동화 + 자재 가격 실시간 연동**.
- **가격 차별화**: 집닥 업체용 월정액 30~80만원 추정 대비 우리는 **49만원 중간 티어** (기능은 더 많음: 견적+자재+사업성 통합).

## [3] Value Proposition Canvas

### 고객 프로필 (오른쪽)

**Customer Jobs** (3가지 층)

- **기능적 Jobs**:
  - 현장 실측 데이터를 표준예가 견적서로 변환 (건당 10시간 → 1분)
  - 자재 가격 변동 시 견적서 자동 업데이트 (재계산 제거)
  - 사업성 평가서 생성 (투자 결정 또는 고객 제안용)
  - 견적서 PDF 전문 디자인으로 출력 (고객 신뢰도 향상)

- **감정적 Jobs**:
  - 엑셀 반복 작업에서 해방되는 느낌 ("드디어 자유다")
  - 견적 오류 불안에서 벗어남 ("이제 안심이다")
  - 고객에게 빠르고 정확한 견적 제시로 전문성 인정받는 느낌 ("프로처럼 보인다")

- **사회적 Jobs**:
  - 동료 업체 대비 "기술 활용하는 혁신적 업체"로 인식되고 싶음
  - 고객이 "이 업체는 체계적이다"라고 평가받고 싶음
  - 집닥 등 플랫폼에서 "빠른 응답 업체" 배지 획득 (경쟁 우위)

**Pains** (반드시 서윤 Pain # 인용)

- **P1**: (Pain #1) 표준예가 견적서 수작업 엑셀 작성으로 건당 10시간 소모 → 수주 기회 3건/월 손실
- **P2**: (Pain #2) 자재 가격 변동(39%) 시 재계산 반복 → 월 15시간 추가 소모 + 견적 오류로 마진 감소
- **P3**: (Pain #18) 표준예가 업데이트 수동 (공공 DB 다운로드) → 시장 가격 미반영으로 경쟁력 상실
- **P4**: (Pain #26) 실측 데이터 입력 후 사업성 자동화 없음 → 투자 결정 지연 (고객 이탈)
- **P5**: (Pain #8) 견적서 표준화 부족 → 고객 비교 시 "비전문적" 인상으로 수주율 하락

**Gains**

- **G1**: 견적 작업 시간 90% 단축 (10시간 → 1시간) → 남는 시간으로 수주 활동 3배 증가
- **G2**: 견적 오류 제로 → 고객 신뢰도 상승 + 재작업 비용 제거
- **G3**: 자재 가격 실시간 반영 → 마진율 15% 개선 (과소 견적 방지)
- **G4**: 전문 디자인 견적서 → 고객 계약 성사율 30% 증가
- **G5**: 사업성 평가서 자동 생성 → 투자 유치 또는 대출 승인 가능성 향상

### 가치 맵 (왼쪽)

**Products & Services**

1. **현장정보 입력 폼** (평수/위치/공간 타입/공정 선택)
2. **표준예가 자동 계산 엔진** (공공 DB 연동: 국토부 표준품셈)
3. **자재 상세내역서 생성** (자재 DB 5만 개 품목, 가격 실시간 API 연동)
4. **사업성 평가서 산출** (예상 원가/마진율/ROI 자동 계산)
5. **견적서 PDF 템플릿** (10종 디자인, 회사 로고 삽입 가능)
6. **자재 가격 알림** (변동 10% 이상 시 푸시 알림)

**Pain Relievers** (각 Pain에 1:1 대응)

- **P1 → 현장정보 입력 1분 → 표준예가 자동 계산 → PDF 다운로드**: 10시간 → 1분 단축 (90% 절감)
- **P2 → 자재 가격 API 실시간 연동 → 견적서 자동 업데이트**: 재계산 제거, 월 15시간 절감
- **P3 → 표준예가 DB 주간 자동 업데이트**: 공공 DB 다운로드 불필요, 월 12시간 절감
- **P4 → 사업성 평가서 자동 생성 (원가/마진/ROI)**: 투자 결정 즉시 가능, 고객 이탈 방지
- **P5 → 전문 디자인 템플릿 10종 제공**: 고객 비교 시 "체계적" 인상, 수주율 30% 증가

**Gain Creators**

- **G1 → 견적 시간 90% 단축**: 남는 시간으로 수주 활동 3배 (월 3건 → 9건)
- **G2 → AI 검증 레이어 (견적 오류 자동 체크)**: 재작업 비용 제로
- **G3 → 자재 가격 실시간 반영**: 마진율 15% 개선 (과소 견적 방지)
- **G4 → 전문 디자인 견적서**: 고객 신뢰도 상승 → 계약 성사율 30% 증가
- **G5 → 사업성 평가서 자동 생성**: 투자 유치 가능성 향상 (은행 대출 승인 근거)

## [4] 공감 지도 (Empathy Map)

**페르소나**: 김시공 (45세 남성, 직원 3인 인테리어 시공업체 대표, 월매출 5천만원, 경기도 부천 기반)

- **Think & Feel** (속으로 무슨 생각/느낌):
  - "견적 작업만 하루 종일 하면 현장 관리는 언제 하나... 직원들한테 미안하다"
  - "엑셀 공식 하나 틀리면 마진 날아가는데, 이거 언제까지 이렇게 하나"
  - "고객이 3개 업체 견적 비교한다는데, 우리가 제일 늦게 보내면 탈락이야"
  - "자재 가격 올랐는지 매번 전화하기 귀찮아... 근데 안 하면 손해"

- **See** (주변에서 뭘 보는가):
  - 집닥 앱에서 경쟁 업체들이 2시간 내 견적 제출하는 걸 봄 (본인은 1~2일 소요)
  - 유튜브에서 "AI 견적 자동화" 광고 보고 관심 생김
  - 네이버 카페에서 동료들이 "엑셀 견적 너무 힘들다" 토로하는 글 매일 봄
  - 고객이 아파트멘터리 같은 "정가 인테리어" 선호하는 트렌드 체감

- **Hear** (동료/상사/고객/가족한테 뭘 듣는가):
  - 동료: "요즘 집닥에서 견적 빨리 안 보내면 리드 안 준대"
  - 고객: "견적서가 너무 복잡해요, 다른 업체는 한 장으로 정리해줬는데"
  - 가족(아내): "당신 주말에도 엑셀만 하네... 애들이랑 놀아줘야지"
  - 자재 공급업체: "가격 올랐어요, 다음 주부터 10% 인상"

- **Say & Do** (겉으로 뭘 하는가):
  - 매주 월요일 오전 3시간 엑셀 견적 작업 (템플릿 복사 → 수정 → 저장)
  - 자재 가격 확인 위해 공급업체 3곳에 전화 (주 2회)
  - 고객에게 "견적서 2~3일 내 보내드릴게요" 말하지만 실제론 5일 걸림
  - 네이버 검색 "인테리어 견적 자동화" (월 1회, 해결책 못 찾음)

- **Pain** (가장 큰 두려움·좌절):
  - **두려움**: "견적 늦게 보내서 수주 놓치면 이번 달 매출 목표 못 채운다"
  - **좌절**: "엑셀 10년 썼는데 아직도 오류 나... 나는 왜 이렇게 느릴까"
  - **분노**: "자재 가격 변동 때문에 견적 다시 하면 시간 2배 걸려, 미치겠다"

- **Gain** ("이것만 해결되면" 이상적 상태):
  - "현장 실측만 하면 견적서가 자동으로 나왔으면... 그럼 하루에 5건도 가능"
  - "자재 가격 변동 알림 오면 견적서 자동 업데이트됐으면 좋겠다"
  - "고객한테 1시간 내 견적 보내서 '이 업체 빠르네' 소리 듣고 싶다"
  - "주말엔 가족이랑 시간 보내고, 평일엔 현장 관리에만 집중하고 싶다"

## [6] 수익 모델 + 4P 마케팅 믹스

### 수익 모델 상세

- **유형**: **Freemium → 월정액 구독** (3단 티어) + **거래망 수수료** (2단계)
  - **근거**: B2B SaaS는 무료 체험 → 유료 전환 모델이 CAC 최소화. 집닥 업체용 월정액 30~80만원 선례 존재 (역기획 Layer 4). 거래망 수수료는 네트워크 효과 발생 시 추가 수익원 (Airbnb 모델).

- **Free Tier**: 
  - **무료 범위**: 견적서 1건/월 생성 가능 (표준예가 자동 계산만, 자재내역·사업성 제외)
  - **제한 기준**: PDF 다운로드 시 워터마크 "리안 SaaS 무료 버전" 표시 + 자재 가격 실시간 연동 불가 (수동 입력만)
  - **목적**: 제품 체험 → "1건 해보니 편하네, 유료 전환" 유도. 워터마크로 고객 제출 시 부담 → 유료 트리거.

- **유료 트리거**: 
  - **사용자 여정 지점**: 무료 1건 생성 후 "2건째 견적 필요 시" Paywall 등장 → "주목 패키지 29만원 구독 또는 건당 5만원 결제" 선택지 제시.
  - **심리 트리거**: "워터마크 제거 + 자재 실시간 연동" 가치 강조. "이번 달 견적 5건 이상이면 구독이 이득" 계산기 제공.

- **가격 앵커링 전략**:
  - **시선 89만원** (최상위 티어)을 먼저 노출 → **집중 49만원**이 "합리적"으로 보임.
  - **주목 29만원**은 "견적만 필요한 소규모"용 → 실제 타겟은 **집중 49만원** (견적+자재+사업성 통합).
  - **비교 앵커**: "엑셀 견적 인건비 월 470만원 vs 우리 49만원 = 10배 ROI" 메시지.

- **경쟁사 가격 대비 포지션**:
  - 집닥 업체용 월정액 30~80만원 추정 → 우리는 **49만원 중간 티어** (기능은 더 많음).
  - 아파트멘터리 B2C 패키지 5,000만원 (25평 기준) → B2B는 없음 → 우리는 **B2B 전용 저가 SaaS**.
  - **포지션**: "집닥보다 기능 많고, 아파트멘터리보다 저렴한 B2B 전용 견적 자동화".

## [7] TAM / SAM / SOM

- **TAM** (전체 시장): 
  - **계산식**: 한국 인테리어 시공업체 5만 개 × 월 평균 견적 비용 50만원 × 12개월 = **3,000억원**
  - **출처**: 서윤 시장조사 "비주거용 인테리어 11.1조원" (상가 7.5조원 + 업무용 3.6조원) 중 견적 자동화 가능 세그먼트 추정 3% = 3,330억원 (보수 추정 3,000억원)

- **SAM** (접근 가능): 
  - **계산식**: 직원 10인 이하 시공업체 4.5만 개 (전체 5만 개 중 90%) × 온라인 SaaS 수용 가능 비율 30% (엑셀 의존도 높은 세그먼트) = 1.35만 개 × 월 49만원 × 12개월 = **793억원**
  - **제약 고려**: 
    - 기술 제약: 표준예가 DB는 "서울·경기 상가"만 지원 (MVP) → 전국 확대는 2단계 → SAM의 50% = **396억원**
    - 지역 제약: 서울·경기 업체 비중 40% (서윤 "경기도 56만호" 근거) → SAM의 40% = **317억원**
    - 언어 제약: 한국어만 지원 (해외 진출 없음)

- **SOM** (초기 확보): 
  - **계산식**: 초기 3개월 유료 구독 50곳 (KR1) × 월 49만원 × 평균 유지 12개월 = **2.94억원** (연 기준)
  - **근거**: 
    - 네이버 카페 5만 명 × 게시글 도달률 10% = 5,000명 노출
    - 랜딩 클릭률 5% = 250명 방문
    - 무료 체험 전환율 20% = 50명 무료 가입
    - 유료 전환율 100% (무료 1건 체험 후 유료 필수) = 50명 유료
  - **6개월 SOM**: 50곳 × 2배 성장 (월 신규 20곳 × 3개월) = 110곳 × 49만원 × 12개월 = **6.47억원**

**데이터 부족 시 표기**: 
- TAM 3,000억원은 "비주거용 인테리어 11.1조원 × 견적 자동화 비중 3%" 추정 — **검증 필요** (실제 견적 비용 비중 불명확).
- SAM 온라인 SaaS 수용 비율 30%는 "엑셀 의존도 높은 세그먼트" 추정 — **검증 필요** (실제 수용도 조사 부재).

## [8] PSF/PMF + GTM 전략

### 현재 단계 판정

- **MVP 출시 시점 판정**: **PSF (Problem-Solution Fit) 단계**
  - **근거**: 
    - Pain Point 검증 완료 (서윤 50개 Pain 중 Level 4~5 통과 13개)
    - Workaround 관찰 완료 (Pain #1, #2, #18 모두 is_workaround_observed=true)
    - 솔루션 가설 설정 완료 (견적 자동화 = Pain Reliever)
    - **아직 PMF 아님**: 실제 사용자 Retention 데이터 없음 (MVP 미출시)

- **PMF 달성 기준**: 
  - **정량 지표**:
    - 재구매율 (구독 갱신율) **≥80%** (월 Churn ≤20%)
    - NPS (Net Promoter Score) **≥50** (추천 의향 80% 이상)
    - 매주 사용률 **≥60%** (주 1회 이상 견적 생성)
    - 유료 전환율 **≥40%** (무료 체험 → 유료 구독)
  - **정성 지표**:
    - "이거 없으면 일 못 해요" 류 고객 피드백 월 10건 이상
    - 자발적 추천 (Referral) 월 5건 이상
    - 경쟁사 전환 (집닥 업체가 우리로 이동) 월 3건 이상

## [9] Flywheel + Engine of Growth

### Flywheel 초안 (6개 요소)

```
1. 견적 시간 단축 체감 → 2. 유료 전환 → 3. 견적 데이터 축적
     ↑                                            ↓
6. 자발적 추천 (Referral) ← 5. NPS 상승 ← 4. AI 정확도 향상
```

**각 연결고리 작동 원리**:
- **1→2**: 무료 1건 체험 시 "10시간 → 1분" 즉시 체감 → "이거 계속 쓰고 싶다" 심리 → 유료 전환율 40% (Paywall 트리거)
- **2→3**: 유료 사용자가 월 평균 10건 견적 생성 → 현장정보·자재 데이터 축적 → DB 10만 건 도달 (6개월)
- **3→4**: 견적 데이터 10만 건 학습 → AI 표준예가 계산 정확도 90% → 95% 향상 (오류율 10% → 5% 감소)
- **4→5**: 견적 오류 감소 → 고객 신뢰도 상승 → "이 업체 견적 정확하네" 평가 → NPS 50+ 달성
- **5→6**: NPS 50+ 사용자가 동료 업체에 "이거 진짜 좋아" 추천 → 자발적 Referral 월 5건 발생 → CAC 0원 고객 획득
- **6→1**: Referral 고객이 무료 체험 → 견적 시간 단축 체감 → 선순환 재시작

### Engine of Growth 선택

**세 가지 중 1개 선택**: **Sticky (리텐션 기반)**

**선택 근거**:
- **Sticky 엔진 특성**: 사용할수록 전환 비용 높아짐 (견적 데이터 축적 → 이탈 시 데이터 손실) + 월 Churn ≤5% 목표 → 장기 LTV 극대화
- **Viral 엔진 배제 이유**: B2B는 바이럴 계수(K) 낮음 (업체끼리 경쟁 관계 → 추천 동기 약함). Referral은 발생하나 주 엔진 아님.
- **Paid 엔진 배제 이유**: CAC 12만원 vs LTV 588만원 → LTV/CAC 49배로 Paid 엔진 가능하나, 초기 예산 제약 (마케팅 월 100만원) → Sticky 우선 후 Paid 확장.

**가속 트리거**: **견적 데이터 10건 이상 축적 시 전환 비용 급증**
- **작동 원리**: 
  - 사용자가 견적 10건 생성 → 과거 프로젝트 데이터 조회 기능 활성화 → "이전 견적 복사 → 수정" 워크플로우 정착
  - 이탈 시 10건 데이터 손실 → 엑셀로 돌아가면 다시 수작업 → 심리적 저항 극대화
  - **데이터**: Dropbox는 사용자가 파일 100개 이상 저장 시 Churn 1% 이하 (선례)

**해자 (Moat) 가능성**:
1. **데이터 해자**: 견적 데이터 10만 건 축적 → AI 정확도 95% (경쟁사 70%) → 신규 진입자 따라잡기 6개월+ 소요
2. **네트워크 효과 (약함)**: 거래망 네트워크 (2단계) 추가 시 협력사 5,000개 확보 → 신규 진입자 네트워크 구축 어려움
3. **전환 비용 해자**: 고객이 3년 견적 데이터 축적 시 이탈 불가능 (데이터 손실 리스크)
4. **브랜드 해자 (없음)**: 초기 단계라 브랜드 인지도 없음 → **실행 속도로 대체** (집닥·아파트멘터리 B2B 모듈 추가 전 6개월 선점)

**해자 없을 시 대응**:
- 집닥이 6개월 내 B2B 견적 모듈 추가 시 → **차별화**: 법률·거래망 통합 (집닥은 견적만) + 데이터 해자 (10만 건 학습 완료)
- 아파트멘터리가 B2B 진입 시 → **가격 경쟁**: 49만원 → 39만원 인하 (마진 희생, 시장 점유율 우선)
- **최악 시나리오**: 두 경쟁사 모두 진입 + 가격 경쟁 심화 → **Exit 전략**: 집닥·아파트멘터리에 M&A 제안 (밸류에이션 50억원, 데이터 자산 기준)
