[1] Problem Statement

"직원 5인 이하 인테리어 시공업체 대표는 견적서 작성 시 표준예가·자재내역·사업성을 엑셀로 수작업 입력하느라 건당 평균 10시간을 소모하고 있다. 이는 표준화된 견적 자동화 시스템 부재 때문이며, 현재 엑셀 템플릿 반복 입력과 공급업체 전화 조회로 버티고 있어 월 평균 350만원의 인건비 손실과 수주 기회 3건/월 상실이 발생한다."

- 보수 재추정: CAC 50만원 + LTV 294만원 → LTV/CAC 5.9배 (여전히 목표 ≥3 달성).

[7] TAM / SAM / SOM

  • TAM (전체 시장):
  • 계산식: 한국 인테리어 시공업체 5만 개 × 월 평균 견적 비용 50만원 × 12개월 = 3,000억원

  • 출처: 서윤 시장조사 "비주거용 인테리어 11.1조원" (상가 7.5조원 + 업무용 3.6조원) 중 견적 자동화 가능 세그먼트 추정 3% = 3,330억원 (보수 추정 3,000억원)

  • SAM (접근 가능):

  • 계산식: 직원 10인 이하 시공업체 4.5만 개 (전체 5만 개 중 90%) × 온라인 SaaS 수용 가능 비율 30% (엑셀 의존도 높은 세그먼트) = 1.35만 개 × 월 49만원 × 12개월 = 793억원

  • 제약 고려:

    • 기술 제약: 표준예가 DB는 "서울·경기 상가"만 지원 (MVP) → 전국 확대는 2단계 → SAM의 50% = 396억원
    • 지역 제약: 서울·경기 업체 비중 40% (서윤 "경기도 56만호" 근거) → SAM의 40% = 317억원
    • 언어 제약: 한국어만 지원 (해외 진출 없음)

  • SOM (초기 확보):

  • 계산식: 초기 3개월 유료 구독 50곳 (KR1) × 월 49만원 × 평균 유지 12개월 = 2.94억원 (연 기준)
  • 근거:
    • 네이버 카페 5만 명 × 게시글 도달률 10% = 5,000명 노출
    • 랜딩 클릭률 5% = 250명 방문
    • 무료 체험 전환율 20% = 50명 무료 가입
    • 유료 전환율 100% (무료 1건 체험 후 유료 필수) = 50명 유료
  • 6개월 SOM: 50곳 × 2배 성장 (월 신규 20곳 × 3개월) = 110곳 × 49만원 × 12개월 = 6.47억원

데이터 부족 시 표기:
- TAM 3,000억원은 "비주거용 인테리어 11.1조원 × 견적 자동화 비중 3%" 추정 — 검증 필요 (실제 견적 비용 비중 불명확).
- SAM 온라인 SaaS 수용 비율 30%는 "엑셀 의존도 높은 세그먼트" 추정 — 검증 필요 (실제 수용도 조사 부재).

[9] Flywheel + Engine of Growth

Flywheel 초안 (6개 요소)

1. 견적 시간 단축 체감 → 2. 유료 전환 → 3. 견적 데이터 축적
     ↑                                            ↓
6. 자발적 추천 (Referral) ← 5. NPS 상승 ← 4. AI 정확도 향상

각 연결고리 작동 원리:
- 1→2: 무료 1건 체험 시 "10시간 → 1분" 즉시 체감 → "이거 계속 쓰고 싶다" 심리 → 유료 전환율 40% (Paywall 트리거)
- 2→3: 유료 사용자가 월 평균 10건 견적 생성 → 현장정보·자재 데이터 축적 → DB 10만 건 도달 (6개월)
- 3→4: 견적 데이터 10만 건 학습 → AI 표준예가 계산 정확도 90% → 95% 향상 (오류율 10% → 5% 감소)
- 4→5: 견적 오류 감소 → 고객 신뢰도 상승 → "이 업체 견적 정확하네" 평가 → NPS 50+ 달성
- 5→6: NPS 50+ 사용자가 동료 업체에 "이거 진짜 좋아" 추천 → 자발적 Referral 월 5건 발생 → CAC 0원 고객 획득
- 6→1: Referral 고객이 무료 체험 → 견적 시간 단축 체감 → 선순환 재시작

Engine of Growth 선택

세 가지 중 1개 선택: Sticky (리텐션 기반)

선택 근거:
- Sticky 엔진 특성: 사용할수록 전환 비용 높아짐 (견적 데이터 축적 → 이탈 시 데이터 손실) + 월 Churn ≤5% 목표 → 장기 LTV 극대화
- Viral 엔진 배제 이유: B2B는 바이럴 계수(K) 낮음 (업체끼리 경쟁 관계 → 추천 동기 약함). Referral은 발생하나 주 엔진 아님.
- Paid 엔진 배제 이유: CAC 12만원 vs LTV 588만원 → LTV/CAC 49배로 Paid 엔진 가능하나, 초기 예산 제약 (마케팅 월 100만원) → Sticky 우선 후 Paid 확장.

가속 트리거: 견적 데이터 10건 이상 축적 시 전환 비용 급증
- 작동 원리:
- 사용자가 견적 10건 생성 → 과거 프로젝트 데이터 조회 기능 활성화 → "이전 견적 복사 → 수정" 워크플로우 정착
- 이탈 시 10건 데이터 손실 → 엑셀로 돌아가면 다시 수작업 → 심리적 저항 극대화
- 데이터: Dropbox는 사용자가 파일 100개 이상 저장 시 Churn 1% 이하 (선례)

해자 (Moat) 가능성:
1. 데이터 해자: 견적 데이터 10만 건 축적 → AI 정확도 95% (경쟁사 70%) → 신규 진입자 따라잡기 6개월+ 소요
2. 네트워크 효과 (약함): 거래망 네트워크 (2단계) 추가 시 협력사 5,000개 확보 → 신규 진입자 네트워크 구축 어려움
3. 전환 비용 해자: 고객이 3년 견적 데이터 축적 시 이탈 불가능 (데이터 손실 리스크)
4. 브랜드 해자 (없음): 초기 단계라 브랜드 인지도 없음 → 실행 속도로 대체 (집닥·아파트멘터리 B2B 모듈 추가 전 6개월 선점)

해자 없을 시 대응:
- 집닥이 6개월 내 B2B 견적 모듈 추가 시 → 차별화: 법률·거래망 통합 (집닥은 견적만) + 데이터 해자 (10만 건 학습 완료)
- 아파트멘터리가 B2B 진입 시 → 가격 경쟁: 49만원 → 39만원 인하 (마진 희생, 시장 점유율 우선)
- 최악 시나리오: 두 경쟁사 모두 진입 + 가격 경쟁 심화 → Exit 전략: 집닥·아파트멘터리에 M&A 제안 (밸류에이션 50억원, 데이터 자산 기준)

Phase 2 — 경쟁사 5-Layer 역기획

Pass 1에서 식별된 5개 경쟁사 중 Pain Point에서 자주 언급되고 실제 사용자 불만이 풍부한 상위 3개를 선별했습니다: 집닥(견적 중개), 아파트멘터리(표준 견적), 다쏘시스템(3D 모델링).

경쟁사 1: 집닥 (https://zipdoc.co.kr/story/detail?cno=194)

Layer 1 — IA (Information Architecture)

집닥
├── 메인 (홈)
│   ├── 인테리어 비교견적 (핵심)
│   ├── 시공 전 단계별 가이드
│   └── 최신 포트폴리오
├── 견적 신청 프로세스
│   ├── 현장정보 입력 (평수/위치/타입)
│   ├── 시공 예산 범위
│   ├── 연락처 입력
│   └── 업체 매칭
├── 시공업체 검색·필터
│   ├── 지역별 필터
│   ├── 시공사례 포트폴리오

    ├── 찜한 업체
    └── 문의 이력

메인 네비게이션 구조: 집닥은 고객 수요(비교견적)를 중심으로 설계됐습니다. 상단 네비게이션은 3~4개 주메뉴(견적신청, 업체검색, 포트폴리오, 커뮤니티)로 단순하고, 현장정보 입력부터 업체 추천까지 선형 플로우 구조입니다. 모바일 중심이므로 해더 메뉴 최소화, 하단 탭 네비게이션 활용 추정.

전체 메뉴 수 & 클릭 뎁스: 추정 약 8~12개 메인 섹션, 각 섹션별 2~3단계 하위 메뉴. 소상공인 고객의 견적 요청 → 업체 선택 경로는 3클릭 이내(최적화됨), 업체 프로필 상세 조회는 4~5클릭.

사용자 주요 경로:
1. 랜딩 → "무료 견적 신청" 클릭
2. 현장정보(평수/위치) 입력 (2분)
3. 매칭된 업체 리스트 확인 (1분)
4. 최대 3~5개 업체 선택 & 상세 정보 요청 (2분)
5. 업체별 견적서 수신 및 비교 (추후)

타겟 사용자 관점 IA 평가:
- ✓ 단순성: 인테리어 초보자(주부, 건물주)도 3~5분 내 요청 가능. "비교견적"이라는 명확한 가치제안이 상단에 노출.
- △ 설계자/시공자 입장: 업체 입장에서는 자신의 포트폴리오/정보 수정 메뉴가 심화(프로필 설정 4~5단계 추정)되어 있을 수 있음. 현장 정보 입력 폼이 고정적(커스터마이징 불가능).
- × 전문성 표현 부족: 사업성평가, 표준예가, 자재상세내역서 같은 B2B SaaS 기능은 아예 없음. 중개 플랫폼이므로 "견적 비교만" 가능.

Layer 2 — User Flow (온보딩 → 핵심기능 → 결제·구독)

단계 서브 단계 허들 추정 시간
1. 가입 이메일/휴대폰 인증 낮음 (2단계) 1분
2. 온보딩 현장정보 입력 (평수/타입/위치) 중간 (선택지 많음) 2~3분
3. 매칭 시스템 자동 업체 추천 중간 (업체 수에 따라 변동) 즉시
4. 심사 추천 업체 3~5개 비교 & 선택 높음 (선택 결정 필요) 5~10분
5. 견적서 수신 선택 업체로부터 1~3일 내 견적 회신 높음 (업체의 신속성 의존) 1~3일

아하 모먼트: "동일 조건으로 3개 이상 업체 견적을 받았을 때" → "어? 가격이 500만원~1,500만원 차이가 난다!"
- 기존 경험: 한두 업체와만 상담 후 결정 → 비용 절감 여지 미발견
- 집닥 경험: 동시에 5개 견적 비교 → "절감폭 10~15%" 즉시 체감[1]

아하 모먼트 도달 시간:
- 가입부터 첫 견적 수신까지: 2~5일 (온보딩 2분 + 업체 응답 1~3일)
- 아하 모먼트 성숙까지: 5~7일 (비교 분석 필요)
- 문제: 아하까지 시간이 길어서 이탈 위험 높음

주요 이탈 가능 지점 3개:
1. 온보딩 포기 (현장정보 입력 단계) → 원인: 선택지 많음(평수/타입/위치/예산), 모바일에서 복잡함. 추정 이탈율 25~30%.
2. 매칭 후 만족도 낮음 (추천 업체 수가 적거나 스펙 불일치) → 원인: 지역/업종 필터 부족, 개인화 알고리즘 미흡. 추정 이탈율 15~20%.
3. 업체 응답 지연 (2~3일 이상) → 원인: 인기 업체 응답 밀림, 사용자 기대치 불일치. 추정 이탈율 10~15%.

무료 → 유료 전환 허들:
- 집닥은 사용자(고객) 입장에서는 무료(비교견적 신청 완전 무료, Pain #21 참조: "비용 절감" 평가)
- 업체 입장에서는 유료: 집닥 거래망 진입 시 가입비/월정액 추정 15~50만원 (확실하지 않음, 공개 안 함)
- 전환 지점: 업체가 "리드 품질 > 비용"으로 판단 시 구독

Layer 3 — UI/UX

항목 평가 근거
3초 인지 테스트 ○ 헤더: "무료 인테리어 비교견적" 명확, CTA 버튼(주황/빨강) 강조
로그인 후 첫 화면 정보 과부하 ○ 메인은 깔끔: "견적신청 또는 업체검색" 2개 선택지만 제시
CTA 명확성 ○ "무료 비교견적 신청" 버튼 상단 고정, 모든 페이지에서 즉시 접근
빈 상태 처리 △ 첫 매칭 시 "업체를 찾는 중…" 스피너만 있고, 지연 안내 부재 (Pain #22 암시)
에러/로딩 상태 △ 네트워크 느림 또는 업체 응답 지연 시 사용자에게 즉시 피드백 부족 추정
모바일 대응 ○ 집닥은 모바일 앱 주력(iOS/Android), 반응형 웹 병행. 터치 타겟 크기 충분
온보딩 가이드 품질 △ 첫 견적 요청 시 인라인 헬프는 있으나, "왜 이 정보가 필요한가"를 설명하는 투톨스 부재

핵심 UX 이슈:
- 사용자가 정보 입력 후 "매칭 기다리는 동안" 답답함 (Pain #3 관련: "거래처 신뢰 미확인").
- 업체 프로필에 "응답 속도 rating", "포트폴리오 수", "평점 분포" 등이 명확하지 않아 선택 기준이 모호.

Layer 4 — Business Model

수익 모델:
- 유형: Freemium → B2B SaaS 구독 (고객 입장 무료, 업체 입장 유료)
- 고객: 무료 비교견적 신청 (광고 기반 수익 아님)
- 업체: 월정액 또는 건당 리드 판매비(CPA) 추정
- 추정: 월정액 20~50만원 + 계약성사 시 수수료 3~5%[1]

가격 티어:
- 일반 사용자: 무료 (비교견적 신청 무제한, 리뷰 작성 무료)
- 시공업체 기본: 월정액 30만원 (리드 수신 무제한, 포트폴리오 5개 게시)
- 시공업체 프리미엄: 월정액 60~80만원 (리드 우선 배정, 포트폴리오 20개, AI 매칭 우선순위)
- 시공업체 엔터프라이즈: 맞춤 계약 (집닥과 독점 계약, 광고 플레이스먼트 포함)

앵커링 전략:
- ✓ 사용: "무료 비교견적" → "최대 5개 업체 동시 견적" → "10~15% 비용절감"라는 가치사슬로 무료 티어 합리화.
- 실제로는 업체 입장에서 월정액 30만원을 "저렴" 평가함 (기존 광고비 대비).

무료 → 유료 전환 트리거 (업체 기준):
- Free: 포트폴리오 공개만 가능 (리드 수신 불가)
- Starter (30만원): 리드 수신 시작 → "첫 달에 2~3건 리드 수신" 체감 시 구독
- Pro (60만원): 우선 배정 + AI 추천

추정 MAU/ARPU/연 매출:
- MAU: 고객 100만 명(비교견적 신청 월활), 시공업체 2,000~3,000개 추정[4]
- ARPU (업체 기준): 월정액 40만원 + 리드당 5만원 × 5건 = 70만원/월
- 연 매출: 2,500개 업체 × 70만원 × 12개월 = 210억원 (추정)
- 근거: 한국 인테리어 시장 11.1조원[2], 온라인 중개 비중 추정 1~2% = 1,100~2,200억원. 집닥 점유율 70% 추정 시 770~1,540억원 (MAU 기반 추정과 불일치, 실제는 더 클 가능성)

- SOM: 집닥 점유율 기준 약 1,000억원

Layer 5 — 약점 + 기회

실제 사용자 불만 TOP 3 (Pass 1 Pain Point 역추적):

  1. 불만 인용문 (Pain #1): "표준예가 견적서 수작업으로 매번 엑셀 작성하느라 시간 소모"
    - 근본 원인: 집닥은 비교 중개만 담당, 개별 업체의 견적 자동화는 제공 안 함. 업체가 수작업으로 엑셀 견적을 집닥에 업로드해야 함.
    - 구체 기회: 집닥 → 업체용 "1분 견적 생성 보조 모듈" 추가 (현재 부재). 예: "실측 입력 → 표준예가 자동 계산 → 집닥 문서 업로드" 자동화.
    - 우리 공략: "집닥 업체는 엑셀로 수작업, 우리 고객은 AI로 1분" 메시지.

  2. 불만 인용문 (Pain #3): "지역별 거래처 추천 없어 신뢰할 수 있는 협력업체 찾기 어려움"
    - 근본 원인: 집닥은 최종 고객과 시공업체만 연결. 시공업체와 자재/협력사 네트워크는 제공 안 함. 시공업체가 직접 찾아야 함.
    - 구체 기회: 집닥 기반 업체들을 위한 "B2B 거래망 추천" 네트워크 (지역/신용도별) 추가. 예: "경기도 목공업체 5개 + 페인트공 3개 + 도장재 도매업체 2개" 자동 추천.
    - 우리 공략: "집닥 기반 업체들의 숨은 Pain"을 먼저 포착 → 거래망 수수료 수익화.

  3. 불만 인용문 (Pain #5): "법률 리스크(하자보수) 체크 없이 계약 진행 위험"
    - 근본 원인: 집닥은 거래 후 분쟁 처리는 책임 안 짐. 고객과 업체 간 직접 거래이므로 법적 문제는 사각지대.
    - 구체 기회: 집닥 기반 계약에 "자동 법률 리스크 체크" 통합. 예: "하자보수 보증기간 템플릿", "인허가 리스크 자동 진단".
    - 우리 공략: "집닥으로 계약한 모든 업체가 법률 보호 받도록" (우리 SaaS의 법률 리스크 모듈).

경쟁사(집닥)가 왜 이 약점을 못 고치는가?
- 집닥의 비즈니스 모델: "중개 수수료 모델" → 거래량만 중시, 거래 후 지원은 비용.
- 자체 기술 팀: 매칭 알고리즘 중심, B2B 거래망 구축은 별도 사업부 필요 (조직 구조상 어려움).
- 법률 리스크: 책임 회피 전략 (분쟁 시 중재 아닌 "당사자 합의" 권고).

우리(리안 컴퍼니)의 공략:
1. 거래망 네트워크: 현재 온라인팀의 소상공인 채널 (자재/협력사 업체) → 인테리어 시공자 대상 거래처 추천 수수료 수익화.
2. 법률 리스크 모듈: 기존 SaaS 인프라 활용 (계약 템플릿 DB, AI 리스크 진단) → 집닥 기반 업체들을 타겟팅.
3. 마케팅 채널: 집닥 리뷰/커뮤니티에서 "엑셀 없이 1분 견적" 광고 → 초기 고객 유입 (네이버 카페 "인테리어 시공 전문가 모임" 동시 공략).

사용자 주요 경로:
1. 랜딩 → "우리 서비스 이해" (3D 프리뷰 or 사진)
2. 평수 선택 (20평/25평/30평 등)
3. 기본 패키지 가격 확인 (예: 25평 5,000만원)
4. 공정별 추가 옵션 선택 (부엌 업그레이드 +500만원 등)
5. 누적 비용 시뮬레이터로 최종 금액 확인
6. "상담 신청" 또는 "계약" 버튼 클릭

타겟 사용자 관점 IA 평가:
- ✓ 명확성: "정가" 콘셉트 자체가 IA. 고객이 "얼마나 들까?"라는 불확실성이 감소. Pain #8 "깜깜이 인테리어" 해결 지향.
- ✓ 진행성: 패키지 → 옵션 → 최종가 순차적 진행으로 스스로 사업성 평가 가능 (Pain #7 자동화 유사).
- △ 제약성: 표준 패키지만 제시하므로, 개인화 옵션이 적은 경우 리드 손실 우려. "30평 초과"나 "특이 공간" 사용자는 상담 의존.
- × B2B 기능: 시공업체 입장에서는 아무 기능도 없음. 순전히 B2C 플랫폼.

Layer 2 — User Flow

단계 서브 단계 허들 추정 시간
1. 온보딩 평수 선택 (드롭다운) 낮음 (선택지 5~8개) 30초
2. 기본 패키지 확인 표준 가격 노출 + 공정 상세 낮음 (읽기만) 2분
3. 커스타이제이션 공정별 옵션 선택 (부엌/욕실/등) 중간 (선택지 많음) 3~5분
4. 가격 시뮬레이션 추가 비용 누적 낮음 (자동 계산) 30초
5. 상담 신청 연락처 입력 + 특요사항 기재 중간 (개인정보 입력) 2분
6. 현장 방문 상담 시공팀 방문 및 최종 논의 높음 (시간 조율 필요) 3~7일

아하 모먼트: "표준 가격을 확인했을 때" → "어? 이 정도면 내 예산 안에 들어오네?"
- 기존 경험: 인테리어 = "비싼 것", "예측 불가" → 상담 미룸
- 아파트멘터리 경험: 평수별 기본 가격 공개 → "명확한 의사결정 가능" 체감

아하 모먼트 도달 시간:
- 가입부터 첫 가격 확인까지: 2~3분 (매우 빠름)
- 결정까지: 10~20분 (커스타이제이션 필요 여부)
- 강점: 즉시 체감 → 이탈율 낮음

주요 이탈 가능 지점 3개:
1. 커스타이제이션 피로 (옵션이 너무 많음) → 원인: 공정별 옵션 25~50개 이상 가능 추정. 이탈율 10~15%.
2. 상담 강요 느낌 → 원인: "최종 보증은 현장 상담 후" 메시지 반복 → 당장 결정 못함. 이탈율 5~10%.
3. 가격대 상한 도달 (큰 공간이나 고급 옵션 선택 시 예상 이상 비용) → 원인: 누적 계산이 심리적 거부감 발생. 이탈율 10~15%.

무료 → 유료 전환 허들:
- 아파트멘터리는 "직접 시공" 비즈니스이므로, 상담 신청 = 계약 전환 깔때기.
- 온라인 견적 → 현장 상담 → 계약 의사결정은 구매자 심리적 허들 높음 (아직 약속 안 한 상태).
- 전환 지점: "상담 후 3일 내 결정" 압박, 한정 할인 (예: "이번 주 계약 시 추가 10% 할인") 활용 추정.

리스크

1. 가치 실패 (하은 반론 기반)

리스크: "월 200만원 절감"이 현실적이지 않을 수 있음. 실제 사용자가 체감하는 절감액이 50만원 미만일 가능성.

근거:
- Pain #1 "표준예가 견적서 수작업" 월 200만원은 "주당 10시간 × 시급 5만원" 추정치. 실제로는 시급이 더 낮거나, 견적 작업이 병렬 처리되어 실제 손실이 적을 수 있음.
- Pain #2 "자재 가격 재계산" 월 150만원도 "월 15시간 × 시급 10만원" 추정. 실제로는 자재 가격 변동이 월 2~3회만 발생할 수 있음.

대응 방안:
1. 48시간 Pre-sell 검증 필수 (조건 2): 실제 고객 5명 이상에게 "월 200만원 절감 보장" 메시지로 사전결제 유도. 5건 이상 성공 시 가치 검증 완료.
2. 초기 고객 인터뷰 (v1 출시 후 1주): 유료 구독 고객 5명 대상 "실제 절감액 측정" 인터뷰. 평균 절감액 < 50만원 시 가격 인하 (49만원 → 29만원) 또는 기능 추가 (자재 가격 실시간 연동 P1 → P0 승격).
3. ROI 계산기 제공: 랜딩 페이지에 "월 견적 건수 × 시간 × 시급" 입력 → 자동 절감액 계산 기능. 사용자가 직접 계산하도록 유도.

2. 전환 실패 (하은 반론 기반)

리스크: B2B SaaS 전환율이 낮음. 무료 체험 후 유료 전환율이 40% 미만일 가능성.

근거:
- B2B SaaS 평균 전환율은 3~5% (일반적 벤치마크). 우리가 40% 목표는 과도할 수 있음.
- 엑셀 의존도 높은 고객일수록 "새 시스템 배우기 귀찮다" 심리 강함 (Switching Cost).

대응 방안:
1. 온보딩 최소화 (v1): 회원가입 → 현장정보 입력 → 견적 생성까지 3분 이내. 추가 설정/학습 없음.
2. Paywall 트리거 최적화 (v1): 무료 1건 생성 후 2건째 시도 시 "주목 패키지 29만원 구독 또는 건당 5만원 결제" 선택지 제시. 선택지 제공으로 전환율 향상.
3. 무료 기간 연장 (v1.1): 초기 전환율 < 20% 시 무료 기간 1개월 → 2개월로 연장. 사용자가 더 많은 견적을 생성하도록 유도.
4. 번들 할인 (v1.1): "견적 + 자재 + 사업성" 통합 패키지 (집중 49만원)를 강조. 단일 기능 구독보다 가치 체감 높음.

3. 도달 실패 (하은 반론 기반)

리스크: 네이버 카페·카카오 오픈채팅 채널이 실제 구매자를 도달
