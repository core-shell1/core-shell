# CDO 분석 — 지역 소상공인 010번호 + 인스타 자동 수집 시스템

## 디자인 비전
"데이터가 말하게 하라" — 수집된 연락처가 주인공. 깔끔한 데이터 테이블과 직관적 진행 상태. 전문가 도구의 신뢰감.

## UX 원칙
1. **입력 → 진행 → 결과 → 저장**: 직선 흐름. 뒤로 가기 없음.
2. **실시간 피드백**: 각 플랫폼별 수집 진행을 시각적으로 보여줌. 기다림의 불안 제거.
3. **데이터 중심 UI**: 화려한 아이콘이나 일러스트 금지. 숫자, 표, 진행 바가 전부.
4. **빠른 재실행**: 과거 이력 클릭 한 번에 동일 조건 재수집.

## 핵심 화면 흐름
메인 입력 화면 → 수집 진행 화면 → 결과 미리보기 화면 → (엑셀 저장) → 이력 화면

## 각 화면 핵심 요소

### 화면1: 메인 입력 화면
**목적**: 30초 안에 수집 시작

**주요 요소**:
- 지역명 입력 필드 (자동완성 지원)
- 업종 선택 (드롭다운 또는 텍스트 입력)
- 플랫폼 5개 체크박스 (네이버플레이스, 카카오맵, 인스타그램, 다이닝코드, 망고플레이트)
  - 기본값: 전체 선택
  - 각 플랫폼 옆에 예상 수집 시간 표시 (예: "약 2분")
- [수집 시작] 버튼 (Primary, 큼직하게)
- 최근 수집 이력 3개 (클릭 시 자동 입력)

**상태 표시**:
- 빈 상태: "지역명을 입력하고 플랫폼을 선택하세요"
- 에러: "지역명을 입력해주세요" (빨간 텍스트, 입력 필드 하이라이트)

**레이아웃**:

### 화면2: 수집 진행 화면
**목적**: 불안 제거 + 진행 상황 실시간 확인

**주요 요소**:
- 전체 진행률 (상단, 큰 프로그레스 바)
- 플랫폼별 진행 카드 5개:
  - 플랫폼명
  - 상태 (대기 중 / 수집 중 / 완료 / 실패)
  - 진행 바
  - 수집된 건수 (실시간 업데이트)
  - 예상 남은 시간
- [중단] 버튼 (Secondary, 우측 상단)

**상태별 표시**:
- 대기 중: 회색, "대기 중..."
- 수집 중: 파란색, 애니메이션, "수집 중... 23건"
- 완료: 초록색, "완료 (47건)"
- 실패: 빨간색, "실패 (에러 메시지)"

**레이아웃**:

### 화면3: 결과 미리보기 화면
**목적**: 수집 결과 확인 + 엑셀 내보내기

**주요 요소**:
- 탭 4개 (전체 / 확인됨 / 미확인 / 폐업의심)
  - 각 탭에 건수 표시 (예: "전체 (152)")
- 데이터 테이블:
  - 컬럼: 체크박스, 업체명, 전화번호, 인스타 ID, 주소, 플랫폼, 수집일시
  - 정렬 가능 (클릭)
  - 행 선택 가능 (체크박스)
- 상단 액션 바:
  - [엑셀 내보내기] (Primary)
  - [선택 항목 내보내기] (Secondary)
  - [다시 수집] (Secondary)
- 하단: 총 건수 요약 (예: "총 152건 / 확인됨 98건 / 미확인 54건")

**빈 상태**:
- "수집된 데이터가 없습니다"
- [다시 수집] 버튼

**레이아웃**:

### 화면4: 이력 화면
**목적**: 과거 수집 재실행 + 비교

  - [결과 보기] 버튼
  - [삭제] 아이콘
- 검색 필드 (지역명/업종 검색)

## 컴포넌트 시스템

### 컬러 (Tailwind 기준)
- **Primary**: blue-600 (수집 시작, 주요 액션)
- **Secondary**: gray-600 (보조 버튼)
- **Success**: green-600 (완료 상태)
- **Warning**: yellow-600 (미확인)
- **Error**: red-600 (실패, 폐업의심)
- **Neutral**: gray-100 (배경), gray-800 (텍스트)

### 핵심 컴포넌트
1. **Input**: 큼직한 입력 필드, 라벨 위치 (지역명, 업종)
2. **Checkbox**: 플랫폼 선택 (체크 시 파란색)
3. **Button**:
   - Primary (파란색, 큼직)
   - Secondary (회색, 작음)
4. **ProgressBar**: 플랫폼별 진행 (애니메이션)
5. **Table**: 정렬 가능, 체크박스, 호버 효과
6. **Card**: 이력 리스트 (그림자, 둥근 모서리)
7. **Tab**: 결과 화면 탭 (언더라인 스타일)
8. **StatusBadge**: 상태 표시 (완료/실패/대기)

### 타이포그래피
- 헤더: 24px, 굵게
- 버튼: 16px, 중간
- 본문: 14px
- 캡션: 12px (플랫폼별 예상 시간)

## 에러 / 로딩 / 빈 상태

### 에러 상태
1. **입력 에러**: 지역명 미입력 시 빨간 테두리 + "지역명을 입력해주세요"
2. **수집 실패**: 플랫폼 카드에 빨간색 + 에러 메시지 (예: "네트워크 오류")
3. **엑셀 저장 실패**: 팝업 알림 "저장 실패. 파일을 닫고 다시 시도하세요"

### 로딩 상태
1. **수집 중**: 플랫폼별 진행 바 + 애니메이션
2. **엑셀 내보내기**: 버튼에 스피너 + "내보내는 중..."

### 빈 상태
1. **메인 화면 최근 이력 없음**: "아직 수집 이력이 없습니다"
2. **결과 테이블 비어 있음**: "수집된 데이터가 없습니다" + [다시 수집] 버튼
3. **이력 화면 비어 있음**: "수집 이력이 없습니다. 첫 수집을 시작하세요"
