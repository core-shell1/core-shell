## User Stories

| ID | As a... | I want to... | So that... | 우선순위 |
|----|---------|-------------|------------|---------|
| US-01 | 영업 담당자 | 전화번호 또는 플레이스 URL만 입력하면 3분 내 제안서가 완성되길 | 하루에 50개 이상 업체를 처리할 수 있도록 | Must |
| US-02 | 영업 담당자 | 경쟁사 대비 우리 업체가 얼마나 뒤처졌는지 숫자로 보여주길 | 사장님에게 "38점 vs 경쟁사 72점"을 근거로 계약 설득 | Must |
| US-03 | 영업 담당자 | 22개 항목 체크리스트와 100점 점수가 자동 계산되길 | 어떤 부분이 문제인지 구체적으로 설명 가능 | Must |
| US-04 | 영업 담당자 | 엑셀에 100개 업체 URL을 넣으면 100개 PPT가 자동 생성되길 | 주말에 배치 돌려서 월요일 영업 출발 | Must |
| US-05 | 영업 담당자 | PPT를 열었을 때 API 키나 기술 정보가 노출되지 않길 | 외부에 공유해도 보안 문제가 없도록 | Must |
| US-06 | 사장님 | 검색 결과 스크린샷으로 실제 내 가게 위치를 보길 | 숫자만 보는 것보다 시각적으로 이해 | Should |
| US-07 | 사장님 | 어떤 키워드로 잠재 고객을 놓치고 있는지 알고 싶음 | 구체적으로 몇 명의 고객을 잃는지 계산된 수치로 확인 | Should |
| US-08 | 영업 담당자 | 크롤링 중 일부 업체 실패해도 전체 작업이 중단되지 않길 | 100개 중 3개 실패해도 97개는 받을 수 있도록 | Must |

## 화면 목록 + 라우트

이 프로젝트는 Python CLI 기반이므로 웹 화면 없음. 대신 입출력 인터페이스 정의.

| 인터페이스 | 파일/경로 | 설명 |
|----------|---------|------|
| 입력: 엑셀 | `샘플/샘플.xlsx` | 업체 URL 목록 (컬럼: URL, 메인 키워드) |
| 출력: PPT | `output/{업체명}_{날짜}.pptx` | 11슬라이드 제안서 |
| 출력: 로그 | 콘솔 stdout | 크롤링 진행 상황 실시간 표시 |
| 설정: .env | `.env` | API 키 (네이버 클라이언트 ID/SECRET) |
